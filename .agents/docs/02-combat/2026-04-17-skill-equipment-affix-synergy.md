# 功能开发文档：技能与装备词条联动

## 1. 基本信息

- 日期：2026-04-17
- 负责人：Codex
- 分类：02-combat
- 关联功能：核心战斗系统优化、职业专精技能系统
- 状态：Testing

## 2. 目标与范围

- 目标：让职业技能与装备词条产生可感知联动，提升技能构筑和装备选择深度。
- 本次范围（In Scope）：
  - 新增技能伤害、主动冷却、触发冷却、技能治疗、元素技能伤害 5 类装备词条。
  - 词条进入装备随机池，并在战斗技能释放时影响伤害、治疗和冷却。
  - 技能面板和属性面板展示相关效果，便于玩家理解搭配收益。
- 非本次范围（Out of Scope）：
  - 不新增技能树分支、主动释放操作或技能升级材料。
  - 不重做装备掉落概率和整体数值曲线。

## 3. 需求与验收标准

- 功能需求：
  - 装备可随机生成技能相关词条。
  - 自动释放技能时，通用技能伤害词条提升所有伤害技能。
  - 元素技能伤害只影响带 `elemental` 标签的技能。
  - 主动/触发冷却缩减按技能类型分别生效，并设置上限避免冷却过低。
  - 技能治疗词条提升技能自带治疗量。
- 验收标准（可检查）：
  - [x] TypeScript 类型检查通过。
  - [x] 生产构建通过。
  - [x] 属性面板可看到技能相关词条累计效果。
  - [x] 技能面板可看到实际技能伤害/冷却/治疗数值。
  - [x] 战斗释放技能时会使用装备词条后的伤害、冷却和治疗。

## 4. 方案设计

- 玩法/交互设计：
  - 稀有及以上装备开始出现技能伤害、主动冷却和技能治疗词条。
  - 传奇及以上装备出现触发冷却和元素技能词条，用于后期构筑分化。
  - 技能详情文案展示“实际伤害百分比、实际冷却秒数、实际治疗百分比”。
- 技术设计：
  - 扩展 `AffixId` 与 `SPECIAL_AFFIXES`。
  - 扩展 `AffixEffects` 聚合字段。
  - 在 `playerUseSkillOnMonster` 计算技能伤害与治疗倍率。
  - 提供 `getEffectiveSkillCooldownMs` 统一计算技能冷却。
- 数据结构与配置：
  - 新词条保留百分比数值，运行时按 `value / 100` 转换。
  - 冷却缩减总上限为 50%，最低有效冷却为 1000ms。
- 兼容性与迁移影响：
  - 旧存档没有新词条时字段默认为 0，不影响现有战斗。
  - 已有装备不迁移，只在新掉落装备上随机出现新词条。

## 5. 实现计划

- 任务拆分：
  - 扩展词条定义和战斗词条聚合。
  - 接入技能伤害、治疗、冷却计算。
  - 更新技能面板、属性面板和功能跟踪文档。
  - 执行类型检查和构建验证。
- 预计改动文件：
  - `src/game/models/affix.ts`
  - `src/game/systems/combat-system.ts`
  - `src/game/scenes/Game.ts`
  - `.agents/docs/07-engine-tech/2026-04-15-features-tracking-migrated.md`
- 风险点与回滚方案：
  - 风险：多件装备叠加导致冷却过短。方案：冷却缩减统一封顶。
  - 风险：词条展示过密。方案：仅在累计值大于 0 时显示。

## 6. 测试计划

- 手工测试步骤：
  - 进入游戏后打开属性面板，确认技能词条累计展示不报错。
  - 装备包含新词条后打开技能面板，确认技能伤害、冷却和治疗展示随词条变化。
  - 进入战斗，确认自动技能释放后冷却 HUD 正常倒计时。
- 边界情况：
  - 无技能词条时保持原技能数值。
  - 多件冷却词条叠加时不低于 1000ms。
  - 非 `elemental` 技能不吃元素技能伤害。
- 性能影响检查：
  - 仅在装备变化和技能释放时执行常量级计算，无额外循环压力。
