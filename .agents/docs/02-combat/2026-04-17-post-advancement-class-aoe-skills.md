# 功能开发文档：转职后职业 AOE 技能

## 1. 基本信息

- 日期：2026-04-17
- 负责人：Codex
- 分类：02-combat
- 关联功能：FEATURES / 职业专精技能系统、核心战斗系统优化
- 状态：Testing

## 2. 目标与范围

- 目标：让角色完成二次转职后，每个职业/专精都拥有具备真实范围命中的 AOE 技能，提升中后期清怪能力和职业差异。
- 本次范围（In Scope）：
  - 设计玩家侧 AOE 技能效果模型
  - 第一阶段为三大基础职业各实现 1 个转职后圆形 AOE 技能
  - 第二阶段扩展为 9 个专精独立 AOE 技能
  - 支持圆形、扇形、直线三种玩家侧 AOE 命中形态
  - 明确自动释放条件、范围、目标上限、伤害倍率和视觉反馈
  - 规划与现有技能槽 `specializationActive`、技能升级、装备技能词条的兼容方式
- 非本次范围（Out of Scope）：
  - 新增复杂地形遮挡、友军单位或持续场地系统
  - 一次性制作独立技能图标和音效资源
  - 重做技能面板交互

## 3. 需求与验收标准

- 功能需求：
  - 转职后自动技能池应提供 AOE 技能选择。
  - AOE 技能命中范围内多个怪物，而不是只命中当前目标。
  - AOE 技能应保留现有自动释放和冷却机制。
  - 不同职业/专精的 AOE 风格应有明显差异。
  - AOE 击杀应复用现有经验、金币、掉落和击杀统计流程。
- 验收标准（可检查）：
  - [ ] 完成二次转职后技能池出现对应 AOE 技能
- [x] AOE 技能可命中半径/扇形/直线范围内多个怪物
  - [ ] 多怪被击杀时，经验、金币、掉落和图鉴击杀记录正确结算
  - [ ] HUD 冷却、技能日志和伤害飘字能显示 AOE 释放结果
  - [ ] 技能伤害词条、元素技能词条、技能升级能影响 AOE 数值
  - [ ] AOE 释放不会导致明显卡顿或重复结算同一只怪

## 4. 方案设计

- 玩法/交互设计：
  - AOE 技能定位为转职后的主力清怪技能，默认放入 `specializationActive` 或作为可替换的专精主动技能。
  - 自动释放条件优先使用 `enemyCountNearby`，避免只有单怪时浪费大范围技能。
  - 三类 AOE 形态：
    - 圆形：围绕玩家或目标点，适合近战压场和法师爆发。
    - 扇形：从玩家朝目标方向展开，适合狂战士正面清怪。
    - 直线/穿透：从玩家到目标路径，适合游侠远程清线。
- 技术设计：
  - 扩展 `SkillEffect`，新增玩家侧 AOE 效果，例如：
    - `aoeDamage`: 半径、倍率、目标上限、中心类型。
    - 后续可扩展 `coneDamage`、`lineDamage`。
  - 扩展技能释放结果，支持多个命中结果：
    - 每个目标的伤害、暴击、是否击杀。
    - 汇总伤害、击杀数、经验、金币。
  - `Game.tryUseClassSkill` 中根据 AOE effect 选择多个怪物目标，并逐个调用伤害结算。
  - 击杀结算复用 `onMonsterKilled`，但需要避免一个技能多次刷新导致流程冲突。
  - 视觉首版使用现有圆形预警/特效能力，玩家技能可显示瞬时范围圈、扇形提示或轨迹线。
- 数据结构与配置：
  - `SkillEffect` 增加 AOE 类型字段。
  - `SkillDefinition.tags` 增加 `aoe`、`cone`、`line`、`area` 等标签。
  - AOE 技能仍使用现有 `cooldownMs`、`priority`、`conditions`、`growth`。
- 兼容性与迁移影响：
  - 旧技能不包含 AOE effect，按现有单体技能流程执行。
  - 旧存档技能 loadout 若没有新 AOE 技能，不需要迁移；转职后可在技能面板手动装备，或后续设计默认替换策略。

## 5. 实现计划

- 任务拆分：
  - 完成技能数据模型设计
  - 设计三职业/九专精 AOE 技能清单
  - 实现玩家 AOE 命中筛选与结算
  - 接入技能 UI、HUD 日志与伤害飘字
  - 接入技能升级和装备词条加成
  - 做多怪击杀回归测试
- 预计改动文件：
  - `src/game/models/skill.ts`
  - `src/game/systems/skill-system.ts`
  - `src/game/systems/combat-system.ts`
  - `src/game/scenes/Game.ts`
  - `.agents/docs/02-combat/2026-04-17-post-advancement-class-aoe-skills.md`
- 风险点与回滚方案：
  - 风险：多怪击杀结算与掉落流程出现重复或漏结。
  - 风险：AOE 过强导致清怪速度和药水消耗失衡。
  - 风险：同屏怪物多时伤害飘字和特效过密。
  - 回滚：保留技能配置但关闭 AOE effect 处理，退回单体 damage 流程。

## 6. 测试计划

- 手工测试步骤：
  - 三职业分别完成转职，确认技能池出现 AOE 技能。
  - 在 1、2、5 个怪物附近释放 AOE，验证命中数量和日志。
  - 用 AOE 同时击杀多个怪，检查经验、金币、掉落、图鉴击杀。
  - 装备技能伤害/元素技能词条后验证伤害变化。
- 边界情况：
  - 范围内没有额外怪物。
  - 当前目标死亡但范围内仍有其他怪物。
  - 多个怪物重叠或刚好在范围边缘。
  - Boss + 小怪同时处于 AOE 范围。
- 性能影响检查：
  - 同屏 20+ 怪物时，AOE 筛选与飘字不能造成明显掉帧。
  - 限制特效对象生命周期，避免残留对象。
