# 功能开发文档：敌人群体行为与感应范围

## 1. 基本信息

- 日期：2026-04-16
- 负责人：Codex
- 分类：02-combat
- 关联功能：FEATURES / 敌人群体行为与感应范围
- 状态：Testing

## 2. 目标与范围

- 目标：为敌人增加感应范围和群体联动逻辑，使一定范围内的敌人能基于玩家位置共同触发靠近或远离策略，提升战斗动态性。
- 本次范围（In Scope）：
  - 敌人拥有独立感应范围
  - 玩家进入范围时触发敌人警戒
  - 感应范围内的敌人可按自身类型执行靠近或远离策略
  - 支持同区域敌人的基础协同行为
- 非本次范围（Out of Scope）：
  - 复杂编队、包围与夹击寻路
  - 敌人之间的仇恨传递网络
  - 声音、受击、技能施法等多种感知源

## 3. 需求与验收标准

- 功能需求：
  - 玩家接近一定范围时，附近敌人应被激活。
  - 被激活敌人应按配置采取靠近或远离策略。
  - 群体行为不能破坏现有刷怪、索敌和战斗结算逻辑。
- 验收标准（可检查）：
  - [ ] 敌人模板支持配置感应范围
  - [ ] 玩家进入范围后，范围内敌人从待机切换到激活
  - [ ] 不同怪物可配置为靠近型或远离型群体响应
  - [ ] 群体激活后行为稳定，不出现整群抖动或瞬移
  - [ ] 玩家离开范围或战斗结束后，敌人状态可恢复或重置

## 4. 方案设计

- 玩法/交互设计：
  - 近战怪更倾向群体压近，远程怪更倾向在感应后拉开到理想距离。
  - 可在怪物图鉴和怪物说明中展示“感应范围”和“行为倾向”。
- 技术设计：
  - 增加敌人感应半径、警戒状态与群体响应触发。
  - 以玩家位置为中心筛选邻近敌人，并批量激活对应 AI 策略。
  - 与现有 `approach` / `retreat` 移动能力复用同一决策层。
- 数据结构与配置：
  - `aggroRadius`
  - `groupBehaviorEnabled`
  - `movementStrategy`
  - `alertState: idle | alerted | engaged`
- 兼容性与迁移影响：
  - 未配置感应范围的旧怪物使用默认值；群体行为默认关闭或使用基础模式。

## 5. 实现计划

- 任务拆分：
  - 扩展怪物模板与运行时状态
  - 接入范围检测与激活逻辑
  - 接入群体响应和移动策略联动
  - 补充调试显示或日志验证
- 预计改动文件：
  - `src/game/models/monster.ts`
  - `src/game/systems/monster-system.ts`
  - `src/game/scenes/Game.ts`
  - `src/game/systems/combat-system.ts`
- 风险点与回滚方案：
  - 风险：范围检测过于频繁，导致性能波动或整群异常联动。
  - 回滚：保留感应范围，仅关闭群体联动。

## 6. 测试计划

- 手工测试步骤：
  - 玩家逐步靠近怪群，观察敌人是否在进入范围后被激活。
  - 混合放置近战与远程怪，确认其策略分化。
  - 退出战斗范围，检查敌人状态是否按设计恢复。
- 边界情况：
  - 玩家同时进入多个怪群的感应范围
  - 怪物被障碍阻挡时的激活行为
  - 远离型怪物被地图边缘限制时的响应
- 性能影响检查：
  - 检查多怪同屏时的范围扫描成本，避免明显帧降。
