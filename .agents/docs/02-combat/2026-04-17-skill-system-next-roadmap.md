- 目标：在已完成的自动技能搭配、触发技能、专精被动和装备词条联动基础上，规划下一阶段技能系统的成长、构筑和职业差异化方向。
- 本次范围（In Scope）：
  - 记录技能升级 / 熟练度、技能符文、标签协同、职业专属机制、触发器扩展、技能连携和技能 UI 完善 7 个后续方向。

  - 明确推荐下一步优先实现“技能升级 / 熟练度系统”。
- 非本次范围（Out of Scope）：

  - 不调整当前技能数值和装备词条概率。
  - 不引入手动释放技能作为当前优先项。

## 3. 需求与验收标准

- 功能需求：
  - 后续技能系统需要提供长期成长目标，而不是只依赖角色等级和装备掉落。
  - 技能搭配需要从“按优先级自动释放”升级为“标签、符文、触发条件之间产生构筑收益”。
  - 三个基础职业需要逐步拥有不同的技能资源或战斗循环。
  - 技能 UI 需要解释基础值、装备加成后值和构筑来源。

## 4. 方案设计

### 4.1 推荐优先级

1. 技能升级 / 熟练度
2. 技能符文 / 技能变体
3. 技能标签协同
4. 职业专属技能机制
5. 技能触发器扩展
6. 技能连携 / Combo
7. 技能 UI 进一步完善

### 4.2 技能升级 / 熟练度

- 玩法目标：
  - 自动释放技能时积累熟练度，让玩家在挂机战斗中持续获得技能成长。
  - 技能等级提升后增强伤害、冷却、治疗、buff 数值或触发阈值。
- 推荐首版：
  - 每个技能记录 `level`、`xp`、`xpToNext`。
  - 释放主动 / 触发技能获得技能经验。
  - 被动技能按战斗时间或击杀结算少量经验。
  - 技能等级先限制为 10 级，降低数值失控风险。
- 数据结构影响：
  - `CharacterData` 增加技能进度字典。
  - `SkillDefinition` 增加成长配置，例如伤害成长、冷却成长、治疗成长。
- 验收标准：
  - 技能释放后技能经验增加。
  - 技能升级后面板显示等级和下一级进度。
  - 技能实际伤害 / 冷却 / 治疗读取等级加成。

### 4.3 技能符文 / 技能变体

- 玩法目标：
  - 让同一个技能能通过符文走不同方向，减少堆叠大量新技能带来的配置成本。
- 示例：
  - `裂地斩`：范围扩大 / 冷却降低 / 斩杀强化。
  - `穿风箭`：暴击强化 / 距离加成 / 连射概率。
  - `奥术冲击`：元素转化 / 护盾联动 / 冷却返还。
- 数据结构影响：
  - 新增 `SkillRuneDefinition`。
  - 技能槽需要记录技能 ID 和符文 ID。
- 验收标准：
  - 每个基础主动技能至少有 2 个符文。
  - 符文能改变技能效果摘要和实际战斗结算。

### 4.4 技能标签协同

- 玩法目标：
  - 让玩家因为标签组合而调整技能搭配，而不是只装最高优先级技能。
- 示例：
  - 装备 2 个 `heal` 技能：治疗溢出转为短时护盾。
  - 装备 2 个 `crit` 技能：暴击后缩短当前自动技能冷却。
  - 装备 `elemental` 技能：元素技能有概率连锁额外伤害。
  - 装备 2 个 `trigger` 技能：触发技能首次释放冷却降低。
- 数据结构影响：
  - 新增 `SkillTagSynergyDefinition`。
  - 自动技能上下文增加当前装备技能标签统计。
- 验收标准：
  - 技能面板显示当前激活的标签协同。
  - 战斗日志能显示标签协同触发结果。

### 4.5 职业专属技能机制

- 玩法目标：
  - 让狂战士、游侠、法师在技能循环上产生明显差异。
- 方向：
  - 狂战士：怒气。受伤、低血或近战命中积累怒气，怒气提升技能伤害或缩短触发技能冷却。
  - 游侠：专注。保持距离、移动或暴击积累专注，专注提升暴击和远程技能收益。
  - 法师：奥术充能。释放技能积累充能，特定技能消耗充能造成爆发或护盾。
- 数据结构影响：
  - `CharacterData` 增加职业资源状态。
  - 战斗结算流程需要在攻击、受击、释放技能、击杀时更新职业资源。
- 验收标准：
  - 三个基础职业各有一个可见职业资源。
  - 至少 1 个技能会消耗或读取职业资源。

### 4.6 技能触发器扩展

- 玩法目标：
  - 增强触发技能的条件表达能力，让自动技能更像可配置战斗策略。
- 可新增触发条件：
  - 暴击后触发。
  - 击杀后触发。
  - 受到伤害后触发。
  - 连续未暴击后触发。
  - 周围敌人死亡后触发。
  - 自身拥有或缺少指定标签 buff 时触发。
- 数据结构影响：
  - 扩展 `SkillCastCondition`。
  - `SkillCastContext` 增加最近一次战斗事件、连续计数和附近死亡事件。
- 验收标准：
  - 至少新增 3 类触发条件。
  - 触发条件可被技能面板文案解释。

### 4.7 技能连携 / Combo

- 玩法目标：
  - 让技能之间产生前后顺序和状态引爆关系，提升构筑深度。
- 示例：
  - 法师 `奥术冲击` 施加“易爆”，`元素耀斑` 命中后引爆额外伤害。
  - 游侠技能施加“破绽”，下一个暴击技能消耗破绽提高暴击伤害。
  - 狂战士技能施加“裂伤”，触发技能命中时按目标已损生命额外结算。
- 数据结构影响：
  - 怪物或战斗目标需要支持短时状态列表。
  - 技能效果新增施加状态、消耗状态、引爆状态。
- 验收标准：
  - 至少实现一套跨技能连携。
  - UI 或战斗日志能展示状态被施加和引爆。

### 4.8 技能 UI 进一步完善

- 玩法目标：
  - 让玩家能读懂技能成长、符文、标签、装备词条和实际生效值。
- 方向：
  - 技能详情弹窗。
  - 显示基础值、等级加成、装备加成、标签协同加成后的最终值。
  - 技能释放日志增加关键来源，例如“元素技能 +18% 生效”。
  - 技能槽显示等级、符文和冷却状态。
- 数据结构影响：
  - 需要统一技能效果摘要生成函数，避免 UI 与战斗计算分叉。
- 验收标准：
  - 技能详情能解释最终伤害、冷却、治疗的来源。
  - 技能槽能显示技能等级和符文。

## 5. 实现计划

- 阶段 1：技能升级 / 熟练度
  - 增加技能经验存档结构。
  - 增加技能释放经验结算。
  - 技能实际数值读取技能等级。
  - 技能面板展示等级和经验。
- 阶段 2：技能符文 / 技能变体
  - 为基础主动技能配置符文。
  - 技能槽支持选择符文。
  - 技能摘要和战斗计算接入符文效果。
- 阶段 3：技能标签协同
  - 增加标签统计与协同配置。
  - UI 显示激活协同。
  - 战斗日志显示触发结果。
- 阶段 4：职业专属技能机制
  - 狂战士怒气、游侠专注、法师奥术充能分职业接入。
- 阶段 5：触发器与连携扩展
  - 扩展技能条件上下文。
  - 给每个职业增加至少一套触发 / 连携玩法。
- 阶段 6：技能 UI 完善
  - 统一技能摘要来源。
  - 展示最终值拆解。

## 6. 测试计划

- 手工测试步骤：
  - 检查技能释放、升级、符文切换、标签协同和触发器扩展互不覆盖。
  - 检查旧存档缺少技能进度、符文或职业资源时能正常补默认值。
  - 检查技能面板和 HUD 在技能信息变多后不溢出。
- 边界情况：
  - 技能达到满级。
  - 技能被卸下后经验保留。
  - 职业切换或专精切换后，不合法技能和符文被清理。
  - 多个冷却缩减来源叠加时仍遵守冷却下限。
- 性能影响检查：
  - 技能经验和标签协同只在释放、击杀或装备变更时结算。
  - 避免每帧全量扫描所有技能和所有状态。
