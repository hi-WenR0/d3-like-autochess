# 功能开发文档：敌人持续缓慢追踪玩家

## 1. 基本信息

- 日期：2026-04-16
- 负责人：Codex
- 分类：02-combat
- 关联功能：FEATURES / 玩家与敌人 AI 移动能力
- 状态：Testing

## 2. 目标与范围

- 目标：让场上敌人持续以较慢速度朝玩家当前位置移动，避免敌人长时间停在原地不动。
- 本次范围（In Scope）：
  - 所有存活敌人在地下城内持续朝玩家方向缓慢移动
  - 玩家移动后，敌人会实时修正追踪方向
  - 探索、战斗、拾取、休息阶段都保持追踪行为
- 非本次范围（Out of Scope）：
  - 绕路、避障、包抄等更复杂寻路
  - 不同怪物类型的差异化追击逻辑
  - 追击动画、朝向或冲刺技能

## 3. 需求与验收标准

- 功能需求：
  - 敌人不再仅在特定战斗目标存在时才移动
  - 敌人始终以低于原始移速的速度靠近玩家
  - 玩家移动到新位置后，敌人能继续朝新位置追踪
- 验收标准（可检查）：
  - [ ] 进入地下城后，场上敌人会持续朝玩家方向移动
  - [ ] 玩家沿任意方向移动时，敌人会改变追踪方向
  - [ ] 敌人移动不会越出地图边界
  - [ ] 不破坏现有索敌、战斗与掉落流程

## 4. 方案设计

- 玩法/交互设计：
  - 追踪速度保持为“缓慢”，保证玩家仍能通过走位拉开空间
  - 保留现有警戒和战斗切换逻辑，持续追踪只负责位移表现
- 技术设计：
  - 在 `Game.ts` 中新增统一敌人持续追踪更新方法
  - 以玩家当前位置作为目标点，逐帧推进敌人坐标
  - 通过速度系数降低追踪速度，避免追击过强
- 数据结构与配置：
  - 先使用场景内常量控制追踪速度系数
  - 暂不新增存档字段
- 兼容性与迁移影响：
  - 仅改运行时移动逻辑，不影响已有存档结构

## 5. 实现计划

- 任务拆分：
  - 新增开发文档并登记需求
  - 抽离敌人持续追踪更新方法
  - 在探索、战斗、拾取、休息阶段接入该逻辑
  - 跑 TypeScript 校验
- 预计改动文件：
  - `src/game/scenes/Game.ts`
  - `.agents/docs/07-engine-tech/2026-04-15-features-tracking-migrated.md`
- 风险点与回滚方案：
  - 风险：敌人贴脸速度过快，压缩玩家操作空间
  - 回滚：仅回退追踪速度系数或恢复到警戒后移动逻辑

## 6. 测试计划

- 手工测试步骤：
  - 进入地下城，观察敌人是否从出生后开始朝玩家移动
  - 玩家沿上下左右移动，观察敌人是否同步修正方向
  - 在拾取和休息阶段观察敌人是否仍继续靠近
- 边界情况：
  - 玩家贴近地图边缘时，敌人仍被正确限制在边界内
  - 多个敌人同时追踪时，不出现明显跳跃
- 性能影响检查：
  - 每帧仅增加轻量方向与位移计算，预期性能影响可忽略
