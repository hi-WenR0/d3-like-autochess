# 功能开发文档：远程普通攻击弹道化

## 1. 基本信息

- 日期：2026-04-16
- 负责人：Codex
- 分类：02-combat
- 关联功能：FEATURES / 主角攻击动作接入
- 状态：Testing

## 2. 目标与范围

- 目标：让游侠与法师的普通攻击改为真实弹道命中后结算伤害。
- 本次范围（In Scope）：
  - 游侠普通攻击发射箭矢
  - 法师普通攻击发射火球
  - 弹道持续追踪目标并在命中时结算伤害
- 非本次范围（Out of Scope）：
  - 技能攻击弹道化
  - 狂战士近战攻击改造
  - 穿透、爆炸、范围伤害

## 3. 需求与验收标准

- 功能需求：
  - 箭矢/火球发射后持续朝敌人修正方向
  - 只有命中敌人时才扣血
  - 命中后弹道立即消失
- 验收标准（可检查）：
  - [ ] 游侠普通攻击使用箭矢图片
  - [ ] 法师普通攻击使用火球图片
  - [ ] 弹道会持续朝向并追踪目标
  - [ ] 命中后敌人才受到伤害，弹道随即消失

## 4. 方案设计

- 玩法/交互设计：
  - 普通远程攻击从即时伤害改为追踪投射物命中伤害
  - 技能仍保持即时结算，减少范围扩大
- 技术设计：
  - 在场景中维护玩家投射物列表并逐帧更新
  - 远程普通攻击发射投射物，不立即结算伤害
  - 命中时调用现有普通攻击结算逻辑
- 数据结构与配置：
  - 不新增存档字段
  - 采用轻量距离阈值命中判定
- 兼容性与迁移影响：
  - 仅影响远程普通攻击流程，对旧存档无影响

## 5. 实现计划

- 任务拆分：
  - 预加载箭矢与火球图片
  - 在 `Game` 中增加投射物运行时管理
  - 修改游侠/法师普通攻击入口为发射投射物
  - 在命中时复用现有伤害与击杀处理
  - 运行 TypeScript 校验
- 预计改动文件：
  - `src/game/player-visuals.ts`
  - `src/game/scenes/Preloader.ts`
  - `src/game/scenes/Game.ts`
- 风险点与回滚方案：
  - 风险：目标死亡或切状态时弹道残留
  - 回滚：恢复远程普通攻击即时结算

## 6. 测试计划

- 手工测试步骤：
  - 游侠普通攻击确认箭矢飞行并命中后扣血
  - 法师普通攻击确认火球飞行并命中后扣血
  - 敌人移动时确认弹道持续转向
  - 目标提前死亡时确认弹道自动消失
- 边界情况：
  - 同时存在多个投射物追踪同一敌人
  - 进城、换层、死亡后场上弹道被正确清理
- 性能影响检查：
  - 仅增加少量玩家投射物逐帧更新，预期无明显性能问题
