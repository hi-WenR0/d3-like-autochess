# 功能开发文档：玩家与敌人 AI 移动策略（靠近 / 远离）

## 1. 基本信息

- 日期：2026-04-16
- 负责人：Codex
- 分类：02-combat
- 关联功能：FEATURES / 玩家与敌人 AI 移动能力
- 状态：Testing

## 2. 目标与范围

- 目标：为玩家和敌人增加基础 AI 移动决策能力，支持“靠近目标”和“远离目标”两种策略，为后续职业、技能、风筝与近战拉扯打基础。
- 本次范围（In Scope）：
  - 角色单位支持 `approach`（靠近）与 `retreat`（远离）两种移动策略
  - 玩家与敌人均可配置当前策略
  - 基于当前目标位置进行实时移动决策
  - 与现有自动战斗循环兼容
- 非本次范围（Out of Scope）：
  - 更复杂的环绕、侧移、保持最优射程等高级走位
  - 多目标威胁评估
  - 地形障碍绕路与寻路系统重构

## 3. 需求与验收标准

- 功能需求：
  - 玩家可根据配置选择靠近或远离当前敌人。
  - 敌人可根据自身配置选择靠近或远离玩家。
  - 移动策略能在战斗/探索过程中驱动单位持续修正位置。
- 验收标准（可检查）：
  - [x] 玩家策略为 `approach` 时，会主动接近当前目标
  - [x] 玩家策略为 `retreat` 时，会主动拉开与当前目标的距离
  - [x] 敌人可独立配置 `approach` / `retreat`
  - [x] 敌我双方策略同时生效时，移动结果稳定且不会抖动异常
  - [x] 不破坏现有自动索敌、战斗结算和拾取流程

## 4. 方案设计

- 玩法/交互设计：
  - 初期可先使用固定策略配置，不强制增加复杂 UI。
  - 后续可将策略与职业、技能、敌人类型绑定。
- 技术设计：
  - 为玩家和怪物引入统一的移动策略枚举，如 `approach | retreat`。
  - 在现有移动更新逻辑中，根据目标方向决定是朝向目标还是反向位移。
  - 增加最小/最大理想距离阈值，避免贴脸抖动和无限逃跑。
- 数据结构与配置：
  - `MovementStrategy = 'approach' | 'retreat'`
  - 玩家运行时字段：`playerMovementStrategy`
  - 敌人运行时或模板字段：`movementStrategy`
- 兼容性与迁移影响：
  - 可先作为运行时字段，不改存档结构；如后续需要持久化再扩展存档。

## 5. 实现计划

- 任务拆分：
  - 定义 AI 移动策略类型与默认值
  - 改造玩家移动逻辑，支持靠近/远离目标
  - 改造敌人移动逻辑，支持靠近/远离玩家
  - 增加基础阈值与防抖处理
- 预计改动文件：
  - `src/game/scenes/Game.ts`
  - `src/game/systems/monster-system.ts`
  - `src/game/models/monster.ts`
- 风险点与回滚方案：
  - 风险：距离判断不稳定导致来回抖动。
  - 回滚：保留 `approach` 为默认行为，仅关闭 `retreat` 分支。

## 6. 测试计划

- 手工测试步骤：
  - 玩家设为靠近，观察是否主动追击目标。
  - 玩家设为远离，观察是否保持拉开趋势。
  - 敌人分别设为靠近/远离，观察是否按策略移动。
- 边界情况：
  - 敌我距离极近时的反向位移
  - 贴近地图边界时的 retreat 行为
  - 无目标状态下的移动逻辑
- 性能影响检查：
  - 仅增加轻量方向计算，不应引起明显帧耗时变化。
