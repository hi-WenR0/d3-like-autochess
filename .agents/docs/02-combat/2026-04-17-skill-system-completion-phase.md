# 功能开发文档：技能系统完善阶段

## 1. 基本信息

- 日期：2026-04-17
- 负责人：Codex

- 关联功能：FEATURES / 职业专精技能系统、核心战斗系统优化

## 2. 目标与范围

- 目标：在自动技能搭配系统首版基础上补齐触发技能、专精被动、技能冷却可读性和技能效果扩展，让 5 槽设计形成完整闭环。
- 本次范围（In Scope）：
  - 每个基础职业新增 1 个触发技能
  - 每个专精新增 1 个专精被动技能
  - 自动补默认基础被动、触发技能和专精被动
  - 支持技能 `buff` 效果并接入临时属性增益
  - HUD 显示当前自动技能冷却状态
  - 技能面板展示标签、触发条件和效果摘要
  - 将可复用的技能判定逻辑抽到技能系统模块
- 非本次范围（Out of Scope）：
  - 技能升级
  - 手动释放技能
  - 复杂召唤物 AI
  - 完整 DOT / 护盾吸收值系统
  - 装备词条按技能标签动态加成

## 3. 需求与验收标准

- 功能需求：
  - `trigger` 槽有可装备且可自动释放的技能。
  - 专精达到 Lv.15 后有可装备专精被动。
  - 技能 buff 可影响角色运行时属性。
  - 玩家能在 HUD 看到技能冷却概况。
  - 技能条件和效果逻辑尽量从 `Game` 场景中收拢。
- 验收标准（可检查）：
  - [x] 狂战士 / 游侠 / 法师各有 1 个基础触发技能
  - [x] 9 个专精各有 1 个专精被动技能
  - [x] 读档或升级后默认技能搭配能自动补齐合法槽位
  - [x] 技能 buff 可改变攻击、防御、攻速、暴击或移速
  - [x] HUD 显示自动技能冷却状态
  - [x] 技能面板能显示技能标签
  - [x] 类型检查与生产构建通过

## 4. 方案设计

- 玩法/交互设计：
  - 触发技能用于处理低血、近身、护盾等自动战斗策略。
  - 专精被动作为 Lv.15 后的构筑补强，占用被动槽形成取舍。
  - HUD 只显示简短冷却概况，详细信息留给技能配置面板。
- 技术设计：
  - 扩展 `SkillEffect.passiveStat` 支持攻速百分比。
  - 扩展 `ActiveBuff` 支持技能来源标识和移速加成。
  - 新增 `skill-system.ts` 承载自动释放条件判断。
  - `Game.ts` 只负责提供上下文、执行动画和结算副作用。
- 数据结构与配置：
  - `sourceId?: string` 标识技能 buff 来源
  - `passiveStat.attackSpeedPct`
  - `SkillCastContext`
- 兼容性与迁移影响：
  - 旧存档读档时重新归一化技能搭配。
  - 不满足等级 / 专精的技能自动从槽位卸下。

## 6. 测试计划

- 手工测试步骤：
  - Lv.13 后确认触发技能自动装备并可释放。
  - Lv.15 且已转职后确认专精被动可装备并生效。
  - 触发技能释放后检查 HUD 冷却变化和日志。
  - 使用技能 buff 后检查属性面板变化。
- 边界情况：
  - 已有旧存档没有触发技能槽内容
  - 低血触发技能与自动药水同时发生
  - buff 已存在时重复触发刷新持续时间
- 性能影响检查：
  - 自动释放条件只在战斗攻击节奏中判断，不做每帧全量扫描。
