# 功能开发文档：主角攻击动作接入

## 1. 基本信息

- 日期：2026-04-16
- 负责人：Codex
- 分类：02-combat
- 关联功能：FEATURES / 主角职业美术资源接入
- 状态：Testing

## 2. 目标与范围

- 目标：为三职业主角接入攻击动作，并保证普通攻击、技能攻击和移动动画之间切换稳定。
- 本次范围（In Scope）：
  - 三职业接入 `Attack` 动画
  - 普通攻击与职业技能都触发 `Attack`
  - 攻击动画播放期间避免被 `idle / walk` 抢占
- 非本次范围（Out of Scope）：
  - 受击与死亡动画
  - 技能专属施法动画
  - 敌人攻击动画

## 3. 需求与验收标准

- 功能需求：
  - 普通攻击和技能攻击都要播放攻击动作
  - 攻击期间方向锁定，避免中途切错方向
  - 攻击结束后平滑回到待机或移动动画
- 验收标准（可检查）：
  - [ ] 三职业都能播放对应方向的 `Attack`
  - [ ] 普通攻击与技能攻击都会触发攻击动作
  - [ ] 攻击期间移动不会打断攻击动画
  - [ ] 攻击结束后能正确回到 `walk` 或 `idle`

## 4. 方案设计

- 玩法/交互设计：
  - 攻击动作仅用于视觉表现，不改变伤害时机和数值
  - 角色可在攻击期间继续移动，但单次攻击的朝向保持锁定
- 技术设计：
  - 扩展主角动画状态为 `idle / walk / attack`
  - 使用时间锁控制攻击动画的持续时间
  - 普通攻击与技能入口统一调用攻击动画触发函数
- 数据结构与配置：
  - 不新增存档字段
  - `Attack` 使用 4 帧、32x32、`U / S / D` 方向资源
- 兼容性与迁移影响：
  - 仅影响运行时视觉状态机，不影响旧存档和战斗结算

## 5. 实现计划

- 任务拆分：
  - 更新主角动画配置与预加载
  - 在 `Game` 中增加攻击动画状态与锁定逻辑
  - 在普通攻击和技能攻击处统一触发
  - 运行 TypeScript 校验
- 预计改动文件：
  - `src/game/player-visuals.ts`
  - `src/game/scenes/Preloader.ts`
  - `src/game/scenes/Game.ts`
- 风险点与回滚方案：
  - 风险：攻击动画与移动状态切换冲突，造成卡住或闪烁
  - 回滚：保留资源预加载，回退攻击锁逻辑

## 6. 测试计划

- 手工测试步骤：
  - 三职业分别触发普通攻击，确认 Attack 播放
  - 三职业分别触发技能，确认 Attack 播放
  - 攻击时按住移动键，确认动画不被 walk 抢占
  - 攻击结束后确认正确回到 walk 或 idle
- 边界情况：
  - 连续攻击时攻击动画可重复刷新
  - 攻击期间快速切换方向，不会中途切错攻击朝向
- 性能影响检查：
  - 仅新增少量动画状态判断，不应产生明显性能问题
