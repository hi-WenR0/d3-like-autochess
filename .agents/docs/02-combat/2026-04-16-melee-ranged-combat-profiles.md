# 功能开发文档：角色与敌人的近战 / 远程战斗属性

## 1. 基本信息

- 日期：2026-04-16
- 负责人：Codex
- 分类：02-combat
- 关联功能：FEATURES / 角色与敌人的近战 / 远程战斗属性
- 状态：Testing

## 2. 目标与范围

- 目标：为角色和敌人增加明确的近战 / 远程战斗属性，用于驱动攻击距离、AI 走位、技能适配与职业差异。
- 本次范围（In Scope）：
  - 玩家与怪物区分近战 / 远程战斗类型
  - 战斗类型影响攻击距离与默认移动策略
  - 为后续职业技能、装备限制、怪物行为扩展提供统一字段
- 非本次范围（Out of Scope）：
  - 完整弹道系统重构
  - 复杂武器切换系统
  - 细分为更多攻击子类型（如法术、投掷、召唤）

## 3. 需求与验收标准

- 功能需求：
  - 角色和敌人都应有可配置的战斗类型字段。
  - 近战单位默认倾向贴近目标，远程单位默认倾向保持射程。
  - 攻击距离、索敌距离和 AI 策略可基于战斗类型派生。
- 验收标准（可检查）：
  - [ ] 角色模型新增近战 / 远程属性
  - [ ] 怪物模型新增近战 / 远程属性
  - [ ] 近战单位攻击距离显著短于远程单位
  - [ ] 远程单位能与现有靠近 / 远离移动策略协同工作
  - [ ] 不破坏现有自动战斗循环与目标锁定流程

## 4. 方案设计

- 玩法/交互设计：
  - 近战单位强调贴身输出与承伤，远程单位强调保持距离与拉扯。
  - 战斗类型可在角色面板、怪物信息与图鉴中显示。
- 技术设计：
  - 为角色与怪物增加统一字段，如 `combatStyle: 'melee' | 'ranged'`。
  - 基于战斗类型计算默认攻击距离、理想距离和移动策略。
  - 让技能、装备、AI 与战斗类型字段解耦接入。
- 数据结构与配置：
  - `CombatStyle = 'melee' | 'ranged'`
  - `attackRange`
  - `preferredCombatDistance`
- 兼容性与迁移影响：
  - 旧存档未包含该字段时，玩家默认近战；怪物按模板默认值补齐。

## 5. 实现计划

- 任务拆分：
  - 扩展角色与怪物模型
  - 接入攻击距离与默认行为计算
  - 调整战斗和移动系统读取新字段
  - 补充 UI 展示与存档兼容逻辑
- 预计改动文件：
  - `src/game/models/character.ts`
  - `src/game/models/monster.ts`
  - `src/game/scenes/Game.ts`
  - `src/game/systems/combat-system.ts`
  - `src/game/systems/monster-system.ts`
- 风险点与回滚方案：
  - 风险：与现有攻击距离阈值冲突，导致单位频繁进退。
  - 回滚：保留字段但仅作为展示，不立即驱动 AI 与攻击判定。

## 6. 测试计划

- 手工测试步骤：
  - 分别创建近战与远程角色，观察攻击触发距离差异。
  - 刷出近战与远程怪物，观察其接敌行为是否不同。
  - 验证切层、回城、读档后战斗类型是否保持正确。
- 边界情况：
  - 远程单位被贴脸后的行为
  - 无法拉开距离时的远程单位攻击判定
  - 多个不同战斗类型单位同时交战
- 性能影响检查：
  - 新增字段应只带来轻量条件判断，不应增加明显帧耗时。
