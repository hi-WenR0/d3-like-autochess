# 功能开发文档：城镇商人 - 自动移动与辅助功能商店

## 1. 基本信息

- 日期：2026-04-17
- 负责人：
- 分类：03-economy
- 关联功能：城镇商人商店与自动移动系统
- 状态：Draft

## 2. 目标与范围

- 目标：
  - 在主城添加商人NPC，提供能力购买服务
  - 允许玩家购买“自动移动能力”和未来扩展的辅助功能
  - 提供自动/手动移动模式切换设置
  - 增强游戏便利性和玩家自定义体验

- 本次范围（In Scope）：
  - 商人NPC设计：位置、外观、交互对话
  - 购买界面：商品展示、价格、购买确认
  - 自动移动能力实现：角色在地牢中自动寻路/探索
  - 移动模式设置：游戏内设置面板切换自动/手动移动
  - 数据持久化：保存玩家已购买能力和当前移动模式
  - 经济平衡：合理的金币价格和游戏进度适配

- 非本次范围（Out of Scope）：
  - 复杂的AI路径规划（使用简单的地牢探索算法）
  - 其他商人商品类型（武器、装备等）
  - 商人任务系统或好感度
  - 自动移动的高级配置（速度、避敌策略等）
  - 移动模式的场景限制（目前所有场景通用）

## 3. 需求与验收标准

- 功能需求：
  1. 主城场景中存在可见的商人NPC
  2. 玩家可以与商人交互打开购买界面
  3. 购买界面显示可购买能力（至少包含“自动移动能力”）
  4. 玩家可以用金币购买能力，购买后永久拥有
  5. 游戏设置中提供自动/手动移动模式切换选项
  6. 自动移动模式下角色能在地牢中自主移动探索
  7. 手动移动模式下玩家使用方向键控制角色
  8. 移动模式设置立即生效且可随时切换

- 验收标准（可检查）：
  - [ ] 进入主城场景，可见商人NPC
  - [ ] 与商人交互，正确打开购买界面
  - [ ] 购买界面显示商品名称、描述、价格
  - [ ] 金币足够时可成功购买，金币减少
  - [ ] 购买后能力列表更新，已购能力标记
  - [ ] 设置面板显示移动模式切换选项
  - [ ] 切换自动移动模式，角色开始自主移动
  - [ ] 切换手动移动模式，角色响应方向键控制
  - [ ] 存档加载后保持已购能力和移动模式设置

## 4. 方案设计

- 玩法/交互设计：
  - 商人位置：主城中心区域，明显标识
  - 交互方式：玩家靠近后按交互键（或点击）打开商店
  - 商店UI：左侧商品列表，右侧详细信息，底部购买按钮
  - 商品内容：
    - 自动移动能力：价格500金币，解锁自动移动功能
    - （预留）未来辅助功能槽位，价格待定
  - 设置入口：游戏内设置面板新增“移动控制”选项卡
  - 移动模式切换：单选按钮“自动移动”/“手动移动”

- 技术设计：
  - 新场景文件：`src/game/scenes/TownMerchant.ts` 处理商人交互和UI
  - 扩展角色数据：`CharacterData` 添加 `purchasedAbilities: string[]` 和 `movementMode: 'auto' | 'manual'`
  - 移动控制系统：修改现有移动逻辑，根据模式选择控制源
  - 自动移动算法：简单的地牢探索（朝最近未探索区域或怪物移动）
  - UI组件：复用现有商店UI组件，扩展商品类型

- 数据结构与配置：
  ```typescript
  // src/game/models/merchant.ts
  export interface MerchantItem {
    id: string;
    label: string;
    description: string;
    price: number;
    requiredAbilities?: string[]; // 前置能力要求
    effect?: { type: 'unlockMovement' | 'other' };
  }

  export const MERCHANT_ITEMS: MerchantItem[] = [
    {
      id: 'auto-movement',
      label: '自动移动能力',
      description: '解锁自动移动功能，角色可在地牢中自主探索',
      price: 500,
      effect: { type: 'unlockMovement' }
    }
  ];
  ```

- 兼容性与迁移影响：
  - 新字段可选，旧存档自动获得默认值：
    - `purchasedAbilities: []`
    - `movementMode: 'manual'`
  - 自动移动能力为付费解锁，不破坏现有手动移动体验
  - 所有现有地牢场景需适配自动移动逻辑

## 5. 实现计划

- 任务拆分：
  1. 设计商人物素图和动画（或使用占位图）
  2. 创建商人NPC类，添加到主城场景
  3. 设计并实现商店UI界面
  4. 扩展角色数据模型，添加购买能力和移动模式字段
  5. 实现购买逻辑和金币扣除
  6. 创建设置面板的移动模式切换UI
  7. 实现自动移动算法（简单探索逻辑）
  8. 修改现有移动控制系统支持模式切换
  9. 测试购买流程和移动模式切换
  10. 平衡金币价格和游戏经济

- 预计改动文件：
  - `src/game/models/character.ts` - 扩展CharacterData
  - `src/game/models/merchant.ts` - 新增商人配置
  - `src/game/scenes/TownScene.ts` - 添加商人NPC
  - `src/game/scenes/TownMerchant.ts` - 新增商人交互场景
  - `src/game/systems/movement-system.ts` - 修改移动控制
  - `src/game/ui/settings-panel.ts` - 新增移动设置
  - `src/game/systems/character-system.ts` - 数据迁移处理

- 风险点与回滚方案：
  - 风险：自动移动AI可能表现不佳（卡住、低效）
    - 缓解：实现简单算法，预留调试开关
    - 回滚：可暂时禁用自动移动，保留购买系统
  - 风险：新字段破坏旧存档
    - 缓解：使用可选字段和默认值处理
    - 回滚：数据迁移函数可恢复默认值
  - 风险：经济平衡失调
    - 缓解：初期设置较高价格，后续根据反馈调整

## 6. 测试计划

- 手工测试步骤：
  1. 新游戏开始，检查主城商人是否出现
  2. 与商人交互，验证商店界面正常打开
  3. 尝试购买自动移动能力（金币足够/不足情况）
  4. 购买后检查能力列表和金币变化
  5. 打开设置面板，切换移动模式
  6. 进入地牢，验证自动移动行为
  7. 切换回手动模式，验证方向键控制
  8. 保存游戏，重新加载验证设置持久化

- 边界情况：
  - 金币为0时尝试购买
  - 重复购买同一能力
  - 切换移动模式时角色正在移动
  - 自动移动时遇到死胡同
  - 自动移动时遭遇战斗
  - 存档迁移（旧存档加载新版本）

- 性能影响检查：
  - 自动移动算法的CPU使用率
  - 商店UI打开时的渲染性能
  - 频繁切换移动模式的响应速度
