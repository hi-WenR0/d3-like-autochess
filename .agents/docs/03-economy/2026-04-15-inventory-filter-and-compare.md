# 功能开发文档：背包高级筛选与装备对比

## 1. 基本信息

- 日期：2026-04-15
- 负责人：Codex
- 分类：03-economy
- 关联功能：FEATURES / 背包高级筛选和装备对比功能
- 状态：In Development

## 2. 目标与范围

- 目标：提升背包整理效率和换装决策效率，支持筛选/排序并在悬浮时给出与当前装备的属性差异。
- 本次范围（In Scope）：
  - 背包按稀有度筛选
  - 背包按部位筛选
  - 背包按稀有度/等级/格位排序（升降序）
  - 悬浮装备时显示与当前已装备物品的属性对比（绿升红降）
- 非本次范围（Out of Scope）：
  - 词条评分系统
  - 多维组合筛选面板重构
  - 背包分页和批量操作改版

## 3. 需求与验收标准

- 功能需求：
  - 玩家可快速切换筛选条件，仅看关心的装备。
  - 玩家可切换排序方式，优先看到高价值装备。
  - 鼠标悬浮背包装备时，显示与当前装备的关键属性差值。
- 验收标准（可检查）：
  - [ ] 背包支持“全部/白/蓝/黄/传说/神话”筛选
  - [ ] 背包支持按部位筛选（含武器/防具/饰品等）
  - [ ] 背包支持排序和升降序切换
  - [ ] Tooltip 显示属性差异，提升为绿色、下降为红色

## 4. 方案设计

- 玩法/交互设计：
  - 在背包面板顶部增加“稀有度/类型/排序”按钮行。
  - 背包格子按筛选结果顺序渲染，不改变真实背包槽位存储。
- 技术设计：
  - 在 `inventory-system.ts` 增加 `queryInventoryItems` 统一筛选+排序逻辑。
  - 在 `Game.ts` 增加背包筛选状态字段并接入 UI 刷新。
  - 扩展 `showTooltip` 支持可选对比装备参数。
- 数据结构与配置：
  - `InventoryQuery`：`rarity`、`slot`、`sortBy`、`sortOrder`
- 兼容性与迁移影响：
  - 不涉及存档结构变更，旧存档直接兼容。

## 5. 实现计划

- 任务拆分：
  - 实现背包查询接口
  - 接入背包筛选/排序 UI
  - 实现装备对比 Tooltip
- 预计改动文件：
  - `src/game/systems/inventory-system.ts`
  - `src/game/scenes/Game.ts`
  - `.agents/docs/07-engine-tech/2026-04-15-features-tracking-migrated.md`
- 风险点与回滚方案：
  - 风险：筛选展示顺序与玩家原始槽位认知不一致。
  - 回滚：可切回“按格位排序”作为默认显示。

## 6. 测试计划

- 手工测试步骤：
  - 分别切换稀有度和部位筛选，检查展示数量。
  - 切换排序方式和升降序，检查顺序变化。
  - 悬浮背包装备，确认出现对比差值且颜色正确。
- 边界情况：
  - 当前筛选结果为空
  - 戒指对比（双戒位）场景
  - 装备穿戴/卸下后面板刷新一致性
- 性能影响检查：
  - 背包 40 格全满时筛选与刷新应无明显卡顿。
