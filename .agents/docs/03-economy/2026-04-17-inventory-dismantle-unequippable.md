# 功能开发文档：背包一键拆解无法装备装备

## 1. 基本信息

- 日期：2026-04-17
- 负责人：Codex
- 分类：03-economy
- 关联功能：FEATURES / 背包拆解功能
- 状态：Testing

## 2. 目标与范围

- 目标：在背包手动拆解功能中增加“一键拆解当前职业无法装备的装备”。
- 本次范围（In Scope）：
  - 增加批量拆解当前职业不可装备装备的系统接口
  - 背包面板增加手动拆解按钮
  - 跳过已锁定装备
  - 保持地牢中禁止手动拆解的限制
- 非本次范围（Out of Scope）：
  - 不改变装备职业限制生成规则
  - 不新增二次确认弹窗
  - 不拆解已装备装备
  - 不拆解锁定装备

## 3. 需求与验收标准

- 功能需求：
  - 玩家在主城背包面板可一键拆解当前职业无法装备的背包装备。
  - 可装备与无职业限制装备不会被拆解。
  - 锁定装备不会被拆解。
- 验收标准（可检查）：
  - [ ] 当前职业无法装备的未锁定装备会被批量拆解
  - [ ] 当前职业可装备装备保留
  - [ ] 无职业限制装备保留
  - [ ] 锁定装备保留
  - [ ] 地牢中点击该功能仍被禁止
  - [ ] 拆解后精华数量正确增加并刷新背包

## 4. 方案设计

- 玩法/交互设计：
  - 在现有“手动拆解”区域增加 `[拆不可用]` 按钮。
  - 点击后直接拆解当前职业不匹配装备，并用战斗日志反馈数量与精华。
- 技术设计：
  - 在 `inventory-system.ts` 增加 `dismantleUnequippable(inventory, baseClass)`。
  - 使用 `getAllowedClassesForEquipment` 判断职业限制；无职业限制视为可装备。
  - 复用现有 `dismantleValue` 精华计算。
- 数据结构与配置：
  - 不新增存档字段。
- 兼容性与迁移影响：
  - 旧存档装备若无 `allowedClasses`，会按现有逻辑视为全职业通用，不会被拆解。

  - 运行 TypeScript 与生产构建校验

- 风险点与回滚方案：
  - 风险：玩家误拆高稀有度但当前职业不可用装备。
  - 回滚：移除按钮与系统接口，恢复原手动拆解入口。

## 6. 测试计划

- 手工测试步骤：
  - 用当前职业准备可装备、不可装备、无职业限制和锁定装备。
  - 在主城点击 `[拆不可用]`，确认只拆不可装备且未锁定装备。
  - 在地牢打开背包点击该按钮，确认禁止拆解。
- 边界情况：
  - 没有不可装备装备。
  - 不可装备装备全部已锁定。
  - 不可装备装备包含不同稀有度。
- 性能影响检查：
  - 仅遍历背包列表，容量固定，性能影响可忽略。
