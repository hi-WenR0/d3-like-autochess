# 功能开发文档：背包自动分解（按装备颜色）

## 1. 基本信息

- 日期：2026-04-15
- 负责人：Codex
- 分类：03-economy
- 关联功能：FEATURES / 背包自动分解与分解系统
- 状态：Testing

## 2. 目标与范围

- 目标：为背包增加自动分解能力，支持按装备等级（颜色/稀有度）自动处理无用装备，并完善分解系统产出闭环。
- 本次范围（In Scope）：
  - 自动分解开关
  - 按颜色阈值配置（如白/蓝/黄）
  - 手动分解与自动分解共用分解规则
  - 分解产物入库与提示反馈
- 非本次范围（Out of Scope）：
  - 装备词条智能评分分解
  - 云端分解策略同步

## 3. 需求与验收标准

- 功能需求：
  - 玩家可配置“自动分解到某稀有度”为开启状态。
  - 掉落进入背包时，满足规则的装备自动分解为材料。
  - 保留手动分解入口，并与自动分解共用材料结算逻辑。
  - UI 显示自动分解状态、分解统计与最近产物。
- 验收标准（可检查）：
  - [x] 支持按颜色设置自动分解阈值（至少白/蓝/黄）
  - [x] 自动分解不会处理已锁定装备
  - [x] 自动分解与手动分解产物一致且可配置
  - [x] 背包与资源面板可看到分解材料变化
  - [x] 存档可持久化自动分解设置

## 4. 方案设计

- 玩法/交互设计：
  - 在背包面板新增“自动分解”区域（开关 + 稀有度下拉）。
  - 新掉落装备判定后即时分解，并弹出轻量日志提示。
- 技术设计：
  - 抽象统一 `dismantle-item` 逻辑，供手动与自动调用。
  - 在背包新增物品入库流程中挂接自动分解判定。
  - 增加装备锁定状态，避免误分解高价值装备。
- 数据结构与配置：
  - `autoDismantleEnabled: boolean`
  - `autoDismantleMaxRarity: EquipmentRarity`
  - `InventoryItem.locked: boolean`（装备锁）
- 兼容性与迁移影响：
  - 老存档默认 `autoDismantleEnabled = false`，不影响原行为。

## 5. 实现计划

- 任务拆分：
  - 增加分解系统核心接口与产物表
  - 接入背包入库流程自动判定
  - 增加背包 UI 设置与状态展示
  - 接入存档读写与迁移默认值
- 预计改动文件：
  - `src/game/systems/inventory-system.ts`
  - `src/game/systems/` 下分解相关系统文件
  - `src/game/ui/inventory-panel.ts`
  - `src/game/systems/save-system.ts`
- 风险点与回滚方案：
  - 风险：误分解导致玩家损失体验。
  - 回滚：自动分解功能可全局开关快速关闭，保留手动分解。

## 6. 测试计划

- 手工测试步骤：
  - 开启自动分解（白/蓝）→ 获取不同颜色装备 → 验证入包或分解结果与材料变化。
- 边界情况：
  - 装备锁定、背包满、连续大量掉落、切场景/重启后的配置恢复。
- 性能影响检查：
  - 高频掉落时，自动分解判定和提示不应引起明显卡顿。
