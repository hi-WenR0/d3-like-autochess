# 功能开发文档：地牢随机事件系统

## 1. 基本信息

- 日期：2026-04-17
- 负责人：Codex
- 分类：01-gameplay-loop
- 关联功能：FEATURES / 地牢随机事件系统
- 状态：Testing

## 2. 目标与范围

- 目标：在地牢探索中加入轻量随机事件，增加楼层过程中的变化和资源决策。
- 本次范围（In Scope）：
  - 每层最多触发一次随机事件
  - 首版提供 5 种事件
  - 事件可直接结算或弹出简单选择
  - 事件结果写入战斗日志并刷新 HUD
- 非本次范围（Out of Scope）：
  - 长篇剧情事件链
  - 复杂多分支对话
  - 事件成就、图鉴或掉落概率大全
  - 独立地图节点系统

## 3. 需求与验收标准

- 功能需求：
  - 玩家在地牢探索阶段有概率遇到随机事件。
  - 同一楼层最多触发一次事件，避免刷屏。
  - 事件至少覆盖收益、风险和选择型交互。
  - 事件不能打断战斗、拾取、休息和楼层结算流程。
- 验收标准（可检查）：
  - [x] 每层最多触发一次随机事件
  - [x] 首版至少有 5 种事件
  - [x] 事件可修改金币、生命、消耗品或临时增益
  - [x] 事件触发时暂停自动探索并显示事件面板
  - [x] 事件结算后恢复探索状态

## 4. 方案设计

- 玩法/交互设计：
  - 随机事件只在地牢探索状态触发。
  - 普通事件提供“确认”按钮，选择事件提供两个结果按钮。
  - 事件结果通过日志提示，不额外引入复杂新面板系统。
- 技术设计：
  - 新增 `dungeon-event` 模型与系统，定义事件配置、触发概率和结果应用。
  - `DungeonState` 增加当前楼层事件触发标记。
  - `Game` 场景在探索状态中按冷却节奏检测事件触发。
- 数据结构与配置：
  - `DungeonEventDefinition`
  - `DungeonEventChoice`
  - `DungeonEventEffect`
  - `DungeonEventContext`
- 兼容性与迁移影响：
  - 旧存档缺少楼层事件标记时默认未触发。

- 风险点与回滚方案：
  - 风险：事件触发频率过高干扰自动战斗。
  - 回滚：降低触发概率或只允许手动进入下一层前触发。

## 6. 测试计划

- 手工测试步骤：
  - 进入地牢并保持探索，确认事件面板可出现。
  - 选择事件结果，确认金币、生命、消耗品或增益变化。
  - 同一层继续探索，确认不会重复触发事件。
  - 进入下一层，确认事件触发机会重置。
- 边界情况：
  - 玩家生命很低时触发伤害事件
  - 背包或消耗品状态变化后触发奖励事件
  - 事件面板打开时怪物、战斗、拾取状态切换
- 性能影响检查：
  - 事件检测使用冷却间隔，不进行每帧复杂扫描。
