# 功能开发文档：地下层完成后的自动存档

## 1. 基本信息

- 日期：2026-04-16
- 负责人：Codex
- 分类：01-gameplay-loop
- 关联功能：FEATURES / 地下层完成后的自动存档
- 状态：Testing

## 2. 目标与范围

- 目标：在每次完成地下层后自动执行存档，降低玩家因中断或异常退出导致的进度损失。
- 本次范围（In Scope）：
  - 地下层完成结算后触发自动存档
  - 自动存档覆盖当前槽位或当前会话档位
  - 提供最小提示，告知本层进度已保存
- 非本次范围（Out of Scope）：
  - 多版本回滚存档
  - 云同步和跨设备存档
  - 存档冲突解决 UI

## 3. 需求与验收标准

- 功能需求：
  - 玩家完成地下层后应自动保存当前进度。
  - 自动存档应包含角色、背包、楼层和关键状态。
  - 自动存档失败时应有可追踪日志或提示。
- 验收标准（可检查）：
  - [ ] 地下层完成事件后自动调用存档
  - [ ] 自动存档写入当前有效存档槽
  - [ ] 重新进入游戏后可从最近已完成楼层继续
  - [ ] 自动存档不会在同一层重复高频触发
  - [ ] 不影响手动存档或主菜单加载流程

## 4. 方案设计

- 玩法/交互设计：
  - 楼层结算后显示短暂“已自动保存”提示，不强制弹窗打断流程。
  - 自动存档应发生在奖励结算完成之后，避免数据缺失。
- 技术设计：
  - 在楼层完成或传送到下一层前的稳定节点统一触发 `saveCurrentSlot`。
  - 增加单层防重复标记，避免同一结算事件多次写盘。
  - 自动存档与手动存档共用同一序列化结构。
- 数据结构与配置：
  - `lastAutoSavedFloor`
  - `autoSaveEnabled`
- 兼容性与迁移影响：
  - 不新增核心存档结构时可直接兼容；如增加标记字段，旧档可补默认值。

## 5. 实现计划

- 任务拆分：
  - 确定楼层完成的统一事件点
  - 接入自动存档调用与节流保护
  - 增加 UI 提示和失败日志
  - 验证与多槽位系统兼容
- 预计改动文件：
  - `src/game/scenes/Game.ts`
  - `src/game/systems/save-system.ts`
  - `src/game/scenes/MainMenu.ts`
- 风险点与回滚方案：
  - 风险：楼层切换过程中状态尚未稳定，导致存档内容不完整。
  - 回滚：只保留提示入口，关闭自动触发。

## 6. 测试计划

- 手工测试步骤：
  - 完成一层地牢后退出游戏，重新加载验证楼层进度。
  - 连续完成多层，检查每层结算后的自动存档是否更新。
  - 人为制造存档失败场景，确认日志或提示可见。
- 边界情况：
  - 楼层完成后立即返回主菜单
  - 自动存档与手动存档几乎同时触发
  - 无有效槽位或旧单槽迁移场景
- 性能影响检查：
  - 楼层完成时允许一次性存档开销，但不能造成明显卡死。
