# 功能开发文档：玩法逻辑优化（主城进入地牢与主城限定装备分解）

## 1. 基本信息

- 日期：2026-04-15
- 负责人：Codex
- 分类：01-gameplay-loop
- 关联功能：FEATURES / 玩法逻辑优化：地牢进入和装备分解限制
- 状态：In Development

## 2. 目标与范围

- 目标：将游戏流程改为“先在主城，再进入地牢”，并限制装备出售/分解只能在主城进行。
- 本次范围（In Scope）：
  - 游戏进入 `Game` 后默认处于主城状态
  - 主城提供“进入地牢”按钮，点击后进入战斗流程
  - 增加主城/地牢状态管理
  - 在地牢状态禁用商店出售逻辑（作为当前装备分解入口限制）
  - 楼层通关后自动回主城并展示本次地牢收益
  - 楼层通关 UI 提供“进入下一层”确认与自动倒计时选项
- 非本次范围（Out of Scope）：
  - 独立 `TownScene` 场景拆分
  - 完整分解系统（材料产出等）实现

## 3. 需求与验收标准

- 功能需求：
  - 启动后先显示主城界面，不直接开战。
  - 玩家点击“进入地牢”后才开始地牢战斗循环。
  - 地牢中禁止装备出售/分解入口。
  - 可从地牢返回主城。
  - 楼层清空后弹出结算 UI 展示本次收益。
  - 结算 UI 可手动进入下一层，且支持“自动进入下一层（3秒倒计时）”。
- 验收标准（可检查）：
  - [ ] 初始进入 `Game` 时处于主城态
  - [ ] 主城界面存在“进入地牢”按钮并可切换到战斗态
  - [ ] 地牢中点击商店会被拦截并提示回城
  - [ ] 回城后可正常打开商店并出售装备
  - [ ] 楼层通关后自动回主城并显示收益列表
  - [ ] 结算 UI 可选择“进入下一层/留在主城”
  - [ ] 勾选自动进入后，出现 3 秒倒计时并自动进下一层

## 4. 方案设计

- 玩法/交互设计：
  - 在 `Game` 场景内加入主城覆盖层（Town Overlay）。
  - 主城覆盖层提供“进入地牢”按钮。
  - HUD 新增“回主城”按钮用于从地牢返回。
- 技术设计：
  - 新增 `gameplayPhase: 'town' | 'dungeon'`。
  - `update()` 仅在 `dungeon` 状态执行战斗循环。
  - 通过 `setWorldVisibility` 控制地牢对象显示。
  - `openShopPanel()` 在 `dungeon` 状态直接拦截。
- 数据结构与配置：
  - 新增运行时状态字段，不改存档结构。
- 兼容性与迁移影响：
  - 老存档可直接加载；加载后先进入主城态。

## 5. 实现计划

- 任务拆分：
  - 增加主城/地牢状态字段与切换方法
  - 增加主城覆盖层 UI 与进入地牢按钮
  - 调整 `update()` 与 HUD 文案
  - 增加商店访问限制
- 预计改动文件：
  - `src/game/scenes/Game.ts`
  - `.agents/docs/07-engine-tech/2026-04-15-features-tracking-migrated.md`
- 风险点与回滚方案：
  - 风险：状态切换时 UI 与对象可见性不一致。
  - 回滚：保留旧流程入口，禁用主城覆盖层。

## 6. 测试计划

- 手工测试步骤：
  - 启动游戏 -> 验证主城界面
  - 点击“进入地牢” -> 验证战斗恢复
  - 地牢中点击商店 -> 验证拦截提示
  - 点击“回主城” -> 主城打开商店并出售装备
- 边界情况：
  - 存档加载后离线奖励弹窗与主城覆盖层的叠加显示
  - 回城时存在 UI 面板的关闭行为
- 性能影响检查：
  - 仅新增少量 UI 与状态判断，不应造成明显帧耗时变化
