# 功能开发文档：玩家方向键移动控制

## 1. 基本信息

- 日期：2026-04-16
- 负责人：Codex
- 分类：01-gameplay-loop
- 关联功能：FEATURES / 玩家方向键移动控制
- 状态：Done

## 2. 目标与范围

- 目标：支持通过上下左右键控制人物移动，补齐基础角色操作方式。
- 本次范围（In Scope）：
  - 上下左右方向键控制角色移动
  - 按住按键时持续移动，松开后停止
  - 与现有战斗、探索和 UI 流程兼容
- 非本次范围（Out of Scope）：
  - WASD 或自定义按键映射
  - 手柄、触屏虚拟摇杆
  - 点击寻路移动

## 3. 需求与验收标准

- 功能需求：
  - 玩家可用上下左右键控制人物移动。
  - 松开按键后，人物停止对应方向移动。
  - 多方向输入时表现稳定，不出现明显卡顿或冲突。
- 验收标准（可检查）：
  - [x] 上下左右键可分别驱动人物移动
  - [x] 按住按键时人物持续移动
  - [x] 松开按键后人物停止移动
  - [x] 与现有场景/战斗流程兼容

## 4. 方案设计

- 玩法/交互设计：
  - 角色以键盘方向输入作为基础移动控制。
  - 保持当前角色移动速度与碰撞逻辑不变。
- 技术设计：
  - 在 `Game` 场景中接入方向键输入监听。
  - 每帧根据当前按键状态计算移动方向向量。
  - 将输入状态限定为运行时数据，不写入存档。
- 数据结构与配置：
  - 不新增持久化数据。
- 兼容性与迁移影响：
  - 旧存档无需迁移，加载后直接支持键盘移动。

## 5. 实现计划

- 任务拆分：
  - 接入方向键输入
  - 将输入映射为角色位移
  - 验证与现有战斗/探索逻辑兼容
- 预计改动文件：
  - `src/game/scenes/Game.ts`
- 风险点与回滚方案：
  - 风险：输入与现有自动行为冲突。
  - 回滚：关闭方向键输入分支，保留原有移动逻辑。

## 6. 测试计划

- 手工测试步骤：
  - 进入游戏后按上下左右键，确认角色移动。
  - 按住方向键，确认角色持续移动。
  - 松开方向键，确认角色停止。
  - 在战斗与非战斗状态下重复测试。
- 边界情况：
  - 同时按两个方向键
  - 快速切换方向键
  - 浏览器窗口失焦后再恢复输入
- 性能影响检查：
  - 方向键输入判断应为轻量逻辑，不应产生可感知性能开销。
