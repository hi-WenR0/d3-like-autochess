## 2. 目标与范围

- 目标：让玩家支持同时按下两个方向键进行 45 度移动，并保证斜向与上下左右移动速度一致。
- 本次范围（In Scope）：
  - 支持 `上+左`、`上+右`、`下+左`、`下+右` 斜向移动
  - 对斜向输入做归一化，避免速度变快
  - 保持原有边界限制与连续按键移动手感
- 非本次范围（Out of Scope）：
  - WASD 输入映射
  - 手柄与触屏虚拟摇杆
  - 冲刺、加速或惯性移动

## 3. 需求与验收标准

- 功能需求：
  - 同时按下两个正交方向键时，人物沿 45 度方向移动
  - 斜向移动总速度与单轴移动一致
  - 冲突输入时保持稳定，不出现异常抖动
- 验收标准（可检查）：
  - [ ] `上+左`、`上+右`、`下+左`、`下+右` 都可正常移动
  - [ ] 斜向移动距离经过归一化后，不会比直线移动更快
  - [ ] 松开其中一个方向键后，可自然切回单方向移动
  - [ ] 不破坏战斗、探索与地图边界逻辑

## 4. 方案设计

- 玩法/交互设计：
  - 保持原有方向键操作方式，仅扩展组合输入能力
  - 单次移动仍使用固定步长，斜向时将步长均分到两个轴
- 技术设计：
  - 不再依赖单一 `manualMoveDirection`
  - 改为根据当前按下的方向键实时生成二维输入向量
  - 当向量长度大于 1 时，按长度归一化后再乘固定步长
- 数据结构与配置：
  - 继续复用运行时按键状态 `manualMovePressed`
  - 不新增存档字段
- 兼容性与迁移影响：
  - 仅影响运行时输入处理，不影响已有存档

- 风险点与回滚方案：
  - 风险：输入状态切换时出现停顿或方向丢失
  - 回滚：恢复为单方向优先的输入解析逻辑

## 6. 测试计划

- 手工测试步骤：
  - 按住任意两个正交方向键，观察人物是否斜向移动
  - 对比斜向和直线移动，确认速度体感一致
  - 松开其中一个方向键，确认人物继续沿剩余方向移动
- 边界情况：
  - 同时按下相反方向键时应相互抵消
  - 贴近地图边缘时斜向移动仍受边界限制
- 性能影响检查：
  - 每帧仅增加一次轻量向量归一化计算，性能影响可忽略
