# 功能开发文档：加载页存档职业显示

## 1. 基本信息

- 日期：2026-04-17
- 负责人：Codex
- 分类：05-ui-ux
- 关联功能：加载存档列表信息展示
- 状态：In Development

## 2. 目标与范围

- 目标：让加载存档页面显示该存档当前职业，而不是固定或错误显示角色名。
- 本次范围（In Scope）：
  - 存档摘要增加当前职业标签
  - 加载页槽位文案改为显示职业
  - 已转职角色优先显示当前专精
- 非本次范围（Out of Scope）：
  - 不修改存档结构
  - 不修改新游戏选角页面

## 3. 需求与验收标准

- 功能需求：
  - 加载页每个有效存档显示该角色当前职业
  - 若角色已专精，则显示专精名称
  - 若未专精，则显示基础职业名称
- 验收标准（可检查）：
  - 不再显示固定“冒险者”
  - 不再显示角色名替代职业
  - 多职业、多专精存档能正确区分显示

## 4. 方案设计

- 玩法/交互设计：
  - 加载页槽位摘要中的名称位改为职业位
- 技术设计：
  - 在 `listSaveSlots()` 中基于存档角色数据解析展示用职业标签
  - 主菜单渲染直接消费该字段
- 数据结构与配置：
  - `SaveSlotSummary` 增加职业显示字段
  - 不新增存档字段
- 兼容性与迁移影响：
  - 旧存档也可通过现有角色数据即时推导职业名称

## 5. 实现计划

- 任务拆分：
  - 新增开发文档
  - 扩展存档摘要数据
  - 更新加载页文本
  - 运行验证
- 预计改动文件：
  - `src/game/systems/save-system.ts`
  - `src/game/scenes/MainMenu.ts`
  - `.agents/docs/05-ui-ux/2026-04-17-save-slot-class-label.md`
  - `.agents/docs/07-engine-tech/2026-04-15-features-tracking-migrated.md`
- 风险点与回滚方案：
  - 风险：旧存档角色字段异常导致职业文案为空
  - 回滚：回退为基础职业标签兜底

## 6. 测试计划

- 手工测试步骤：
  - 打开加载页，确认不同存档显示不同职业
  - 已专精存档显示专精名
  - 未专精存档显示基础职业名
- 边界情况：
  - 异常存档仍能显示兜底职业文案
  - 空存档槽位文案不受影响
- 性能影响检查：
  - 存档列表渲染无明显变化
