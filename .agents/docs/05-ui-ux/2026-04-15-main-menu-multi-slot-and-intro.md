- 负责人：Codex

## 2. 目标与范围

- 目标：增强主菜单可用性，支持多存档槽管理与游戏介绍页面，提升开局与回流体验。
- 本次范围（In Scope）：
  - 主菜单提供“新游戏 / 加载游戏 / 游戏介绍”入口
  - 加载游戏界面展示 20 个存档槽
  - 支持从槽位加载和删除存档
  - 支持游戏介绍页面与返回主菜单
- 非本次范围（Out of Scope）：
  - 云存档同步
  - 存档截图缩略图
  - 多语言介绍页

## 3. 需求与验收标准

- 功能需求：
  - 玩家可从主菜单创建新游戏。
  - 玩家可在 20 个槽位中选择加载或删除存档。
  - 玩家可查看游戏介绍并返回。
- 验收标准（可检查）：
  - [x] 主菜单包含三入口并支持点击跳转
  - [x] 加载界面可显示 20 槽位状态（空/有存档）
  - [x] 已有存档槽可加载进入游戏
  - [x] 已有存档槽可删除并刷新显示
  - [x] 游戏介绍页面可正常打开与返回

## 4. 方案设计

- 玩法/交互设计：
  - 主菜单作为入口总控，加载和介绍可作为子页面或独立场景。
  - 存档槽显示摘要：角色名、等级、楼层、时间戳。
- 技术设计：
  - 将现有单存档键扩展为 `save_slot_01 ... save_slot_20`。
  - 在 `save-system.ts` 增加按槽位读写、枚举、删除接口。
  - 在 `MainMenu.ts` 增加菜单状态机或拆分为多个场景组件。
- 数据结构与配置：
  - `slotId: number (1-20)`
  - `SlotSummary`: `{ slotId, empty, timestamp, level, floor, name }`
- 兼容性与迁移影响：
  - 旧单存档可迁移到槽位 1；无法迁移时保持空槽并提示。

## 5. 实现计划

- 任务拆分：
  - 扩展存档系统支持 20 槽位操作
  - 重构主菜单 UI 与页面切换
  - 实现加载页槽位列表和删除按钮
  - 实现游戏介绍页
- 预计改动文件：
  - `src/game/scenes/MainMenu.ts`
  - `src/game/systems/save-system.ts`
  - `src/game/main.ts`（必要时新增场景注册）
- 风险点与回滚方案：
  - 风险：旧存档兼容导致读取异常。
  - 回滚：保留单槽读取 fallback 逻辑。

## 6. 测试计划

- 手工测试步骤：
  - 启动进入主菜单，逐项点击三入口。
  - 创建新游戏后存档，检查加载页是否显示对应槽位信息。
  - 删除槽位后刷新，确认状态变为空。
  - 打开游戏介绍页并返回主菜单。
- 边界情况：
  - 20 槽全空、20 槽全满、删除后立即加载、无效槽位访问。
- 性能影响检查：
  - 菜单页面切换与 20 槽渲染应保持流畅。
