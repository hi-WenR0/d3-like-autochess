# 功能开发文档：怪物图鉴

## 1. 基本信息

- 日期：2026-04-16
- 负责人：Codex
- 分类：05-ui-ux
- 关联功能：FEATURES / 怪物图鉴
- 状态：Testing

## 2. 目标与范围

- 目标：增加怪物图鉴功能，记录玩家已击败过的怪物信息、击败数量和说明，强化收集感与内容认知。
- 本次范围（In Scope）：
  - 解锁已击败怪物的图鉴条目
  - 展示怪物名称、说明、击败数量与基础特征
  - 图鉴数据持久化到存档
- 非本次范围（Out of Scope）：
  - 怪物掉落大全和概率透出
  - 动态 3D/骨骼展示
  - 图鉴成就与奖励联动

## 3. 需求与验收标准

- 功能需求：
  - 玩家击败过的怪物会解锁图鉴。
  - 图鉴中能查看说明、累计击败数量等信息。
  - 图鉴应与怪物模板数据和存档系统集成。
- 验收标准（可检查）：
  - [ ] 击败新怪物后生成或解锁图鉴条目
  - [ ] 图鉴可显示怪物名称、简介、击败数量
  - [ ] 重新读档后图鉴进度保持
  - [ ] 未解锁怪物显示隐藏态或占位态
  - [ ] 图鉴界面可从主菜单或游戏内入口打开

## 4. 方案设计

- 玩法/交互设计：
  - 已解锁怪物显示完整信息，未解锁怪物显示剪影或问号。
  - 条目可补充近战 / 远程、感应范围、常见掉落等基础标签。
- 技术设计：
  - 增加图鉴数据结构，按怪物模板 ID 记录是否解锁和击败次数。
  - 在怪物死亡结算时更新图鉴进度。
  - 增加图鉴面板并接入现有 UI 管理系统。
- 数据结构与配置：
  - `MonsterCodexEntry = { monsterId, unlocked, killCount, lastSeenAt }`
  - 图鉴展示字段复用怪物模板中的 `name`、`description`、`tags`
- 兼容性与迁移影响：
  - 旧存档缺少图鉴数据时，以空图鉴初始化。

## 5. 实现计划

- 任务拆分：
  - 定义图鉴数据模型与存档结构
  - 在怪物击败事件中更新图鉴
  - 实现图鉴 UI 面板与入口
  - 补充怪物说明配置
- 预计改动文件：
  - `src/game/models/monster.ts`
  - `src/game/systems/monster-system.ts`
  - `src/game/systems/save-system.ts`
  - `src/game/scenes/MainMenu.ts`
  - `src/game/scenes/Game.ts`
- 风险点与回滚方案：
  - 风险：怪物模板描述信息不足，导致图鉴内容过空。
  - 回滚：先上线基础统计版，仅展示名称和击败数量。

## 6. 测试计划

- 手工测试步骤：
  - 击败一种新怪物，确认图鉴解锁并显示计数为 1。
  - 连续击败同种怪物，确认计数累加。
  - 存档、退出、重新加载后核对图鉴数据。
- 边界情况：
  - 同一帧击败多个同种怪物
  - 未配置描述字段的怪物
  - 图鉴中存在旧版本已删除怪物 ID
- 性能影响检查：
  - 图鉴更新为轻量计数写入，不应影响战斗结算流畅度。
