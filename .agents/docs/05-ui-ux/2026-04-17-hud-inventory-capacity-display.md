# 功能开发文档：HUD 背包容量显示

## 1. 基本信息

- 日期：2026-04-17
- 负责人：Codex
- 分类：05-ui-ux
- 关联功能：HUD 信息展示
- 状态：Done

## 2. 目标与范围

- 目标：在游戏界面 HUD 中，金币下方显示背包占用，例如 `背包: 111/200`。
- 本次范围（In Scope）：
  - 在金币文本下方新增背包容量文本
  - 使用粉红色字体突出显示
  - 避免与现有战斗日志重叠
- 非本次范围（Out of Scope）：
  - 背包系统逻辑调整
  - HUD 整体重排
  - 背包容量扩展机制

## 3. 需求与验收标准

- 功能需求：
  - 显示当前背包已用格数与总容量
  - 文本位于金币下方
  - 文本颜色为粉红色
- 验收标准（可检查）：
  - [x] HUD 左侧信息区新增 `背包: 已用/总量`
  - [x] 总容量取自 `INVENTORY_CAPACITY`
  - [x] 文本不会和战斗日志重叠

## 4. 方案设计

- 玩法/交互设计：
  - 让玩家在主界面直接感知背包剩余空间
- 技术设计：
  - 为 `Game` 增加 `inventoryUsageText`
  - 在 `renderHUD()` 创建文本，在 `updateHUD()` 更新占用数
  - 将战斗日志与技能状态文本整体下移一行
- 数据结构与配置：
  - 不改存档
  - 总容量直接使用现有常量 `INVENTORY_CAPACITY`
- 兼容性与迁移影响：
  - 仅影响 HUD 显示层

## 5. 实现计划

- 任务拆分：
  - 新增 HUD 背包容量文本
  - 更新 HUD 刷新逻辑
  - 调整日志文本位置避免遮挡
  - 运行 TypeScript 与构建验证
- 预计改动文件：
  - `src/game/scenes/Game.ts`
  - `.agents/docs/05-ui-ux/2026-04-17-hud-inventory-capacity-display.md`
  - `.agents/docs/07-engine-tech/2026-04-15-features-tracking-migrated.md`
- 风险点与回滚方案：
  - 风险：HUD 局部文本密度提高
  - 回滚：移除容量文本并恢复日志位置

## 6. 测试计划

- 手工测试步骤：
  - 进入主城与地牢，确认金币下方显示背包占用
  - 拾取装备后，确认数字实时增长
  - 接近满包时，确认显示如 `背包: 199/200`
  - 观察战斗日志与技能状态是否仍清晰可见
- 边界情况：
  - 空背包显示 `背包: 0/200`
  - 满背包显示 `背包: 200/200`
- 性能影响检查：
  - 仅增加一行 HUD 文本刷新，影响可忽略
