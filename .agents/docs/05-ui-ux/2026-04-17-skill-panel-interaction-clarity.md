# 功能开发文档：技能面板交互清晰化

## 1. 基本信息

- 日期：2026-04-17
- 负责人：Codex
- 分类：05-ui-ux
- 关联功能：FEATURES / 职业专精技能系统、自动技能搭配系统
- 状态：Testing

## 2. 目标与范围

- 目标：优化技能配置面板的交互方式，让玩家更清楚“当前装备了什么技能、哪些技能可装备、点击哪里装备、为什么不可装备”。
- 本次范围（In Scope）：
  - 重新设计技能面板的信息层级
  - 明确技能槽与技能池的交互关系
  - 设计可装备 / 已装备 / 不可装备 / 未解锁状态
  - 设计技能详情、筛选和装备反馈
  - 保持技能仍为自动释放，不增加战斗中手动施法
- 非本次范围（Out of Scope）：
  - 不重做技能系统底层规则
  - 不新增技能树或技能符文
  - 不制作独立图标资源
  - 不改变现有存档结构

## 3. 需求与验收标准

- 功能需求：
  - 玩家打开技能面板后，能一眼看出当前 5 个技能槽。
  - 玩家能清楚知道每个技能可装备到哪些槽位。
  - 技能池应支持按槽位 / 类型 / 状态筛选或分组。
  - 点击技能或槽位后的下一步操作必须明确。
  - 不可装备技能需要展示原因。
- 验收标准（可检查）：
  - [x] 当前技能槽区域清楚展示槽位、技能名、等级、触发条件和冷却
  - [x] 技能池中已装备技能有明确标记
  - [x] 技能池中不可装备技能有明确原因
  - [x] 装备技能不再依赖多个小型 `[槽位]` 文本按钮作为唯一入口
  - [x] 玩家可以通过“选择槽位 -> 选择技能”或“选择技能 -> 装备到槽位”的明确流程完成配置
  - [x] 技能较多时面板不溢出，信息仍可读

## 4. 方案设计

- 玩法/交互设计：
  - 技能仍全部自动释放；面板只负责配置自动技能搭配。
  - 推荐交互主线：先选择左侧技能槽，右侧技能池只显示或高亮可装备到该槽的技能。
  - 右侧技能卡点击后，如果当前已选槽位且技能兼容，则直接装备；否则在详情区域显示可装备槽位按钮。
  - 已装备技能显示“已装备：槽位名”，避免重复装备时困惑。
- 技术设计：
  - 在 `Game.openSkillLoadoutPanel()` 内增加本地 UI 状态字段，例如 `selectedSkillSlot` / `selectedSkillId`。
  - 保持现有 `equipSkill()`、`canEquipSkill()`、`isSkillUnlocked()` 作为唯一规则来源。
  - 将技能池渲染拆成：
    - 槽位栏
    - 技能列表
    - 技能详情 / 操作区
  - 可先不引入滚动列表组件，第一阶段通过分组、分页或只显示相关技能控制数量。
- 数据结构与配置：
  - 不新增存档字段。
  - 仅新增运行时 UI 选择状态。
- 兼容性与迁移影响：
  - 不影响旧存档。
  - 已装备技能继续读取现有 `character.skillLoadout`。

  - 回滚：保留底层技能配置，恢复旧面板渲染。

## 6. 测试计划

- 手工测试步骤：
  - 新角色打开技能面板，确认默认基础主动可理解。
  - 转职后打开技能面板，确认专精主动和 AOE 技能可理解并可装备。
  - 选择不同槽位，确认右侧技能池状态变化正确。
  - 装备 / 替换 / 重复装备技能，确认反馈和槽位结果正确。
- 边界情况：
  - 未转职角色查看专精技能。
  - 技能池技能很多，包含已装备、可装备、未解锁、不可装备状态。
  - 被动技能两个槽位互斥，不能重复装备。
- 性能影响检查：
  - 技能面板打开时仅渲染有限数量技能卡，避免对象过多。
