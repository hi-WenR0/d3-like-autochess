# 功能开发文档：主角职业 spritesheet 美术接入

## 1. 基本信息

- 日期：2026-04-16
- 负责人：Codex
- 分类：05-ui-ux
- 关联功能：FEATURES / 主角职业美术资源接入
- 状态：Testing

## 2. 目标与范围

- 目标：将三种职业的主角从代码方块替换为 spritesheet 角色美术。
- 本次范围（In Scope）：
  - 三职业主角接入待机与行走动画
  - 根据移动方向切换上、下、侧向动画
  - 左向复用侧向资源并使用 `flipX`
- 非本次范围（Out of Scope）：
  - 攻击、受击、死亡动画接入
  - 敌人美术资源替换
  - 主菜单或 UI 头像资源替换

## 3. 需求与验收标准

- 功能需求：
  - 狂战士、游侠、法师进入游戏后显示对应职业角色美术
  - 玩家移动时播放行走动画，停止时播放当前朝向待机动画
  - 斜向移动保持现有速度逻辑，并切到合理方向动画
- 验收标准（可检查）：
  - [ ] 三职业都能正确显示对应 spritesheet
  - [ ] 上下移动分别切到 `U_*` / `D_*` 动画
  - [ ] 左右移动切到 `S_*`，左向使用翻转
  - [ ] 停止移动后切回最后朝向的待机动画

## 4. 方案设计

- 玩法/交互设计：
  - 保留当前键盘移动方式，仅替换主角视觉表现
  - 不改变现有地牢 HUD、战斗和掉落流程
- 技术设计：
  - 在 `Preloader` 中预加载三职业的 `Idle` / `Walk` spritesheet
  - 在 `Game` 中创建职业动画并根据输入向量更新动画状态
  - 使用独立配置文件统一管理职业目录、动画 key、帧数和缩放
- 数据结构与配置：
  - 不新增存档字段
  - 使用运行时常量映射职业与资源目录
- 兼容性与迁移影响：
  - 旧存档加载后会根据已有 `baseClass` 自动显示对应职业资源

## 5. 实现计划

- 任务拆分：
  - 建立主角美术共享配置
  - 预加载三职业待机/行走 spritesheet
  - 替换主角矩形渲染为 sprite 动画
  - 接入朝向与翻转逻辑
  - 运行 TypeScript 校验
- 预计改动文件：
  - `src/game/player-visuals.ts`
  - `src/game/scenes/Preloader.ts`
  - `src/game/scenes/Game.ts`
- 风险点与回滚方案：
  - 风险：sprite 缩放或锚点不合适，导致角色位置与文字偏移异常
  - 回滚：恢复到原来的矩形主角渲染逻辑

## 6. 测试计划

- 手工测试步骤：
  - 分别使用三职业开新档，确认角色显示正确
  - 测试上下左右与斜向移动的动画切换
  - 停止移动后观察是否回到正确待机动画
- 边界情况：
  - 玩家贴墙移动时动画状态保持稳定
  - 进城、回地牢、楼层切换后角色 sprite 仍正常显示
- 性能影响检查：
  - 仅新增少量主角动画播放，不应造成明显性能问题
