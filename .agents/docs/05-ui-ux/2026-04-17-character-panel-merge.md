# 功能开发文档：主界面角色面板合并

## 1. 基本信息

- 日期：2026-04-17
- 负责人：Codex
- 分类：05-ui-ux
- 关联功能：主界面角色属性栏与装备栏整合
- 状态：In Development

## 2. 目标与范围

- 目标：将主界面原本分离的“装备栏”和“角色属性栏”合并为一个“角色”面板，减少入口分散。
- 本次范围（In Scope）：
  - 底部导航入口改为单个“角色”按钮
  - 合并原属性面板与装备面板内容
  - 左侧显示角色属性与可分配属性点
  - 右侧显示装备栏与装备总加成
- 非本次范围（Out of Scope）：
  - 不调整背包、技能、商店等其他面板结构
  - 不改装备逻辑、属性计算逻辑

## 3. 需求与验收标准

- 功能需求：
  - 玩家点击“角色”后打开一个统一面板
  - 左边保留原角色属性信息
  - 右边保留原装备栏信息
- 验收标准（可检查）：
  - 底部导航不再单独显示“装备”“属性”，改为“角色”
  - 角色面板可正常关闭、拖动
  - 装备悬浮提示与卸装逻辑仍可用
  - 属性点分配后面板可正确刷新

## 4. 方案设计

- 玩法/交互设计：
  - 使用单一弹窗承载角色相关信息
  - 左右双栏布局，避免信息来回切换
- 技术设计：
  - 新增 `openCharacterPanel()` 统一构建 UI
  - `openEquipPanel()` 与 `openStatsPanel()` 保留为兼容入口并重定向到新面板
- 数据结构与配置：
  - 不新增存档字段
  - 复用现有角色属性、装备和词条数据
- 兼容性与迁移影响：
  - 原有按钮行为改为进入新面板
  - 其他调用原方法的位置仍可正常工作

## 5. 实现计划

- 任务拆分：
  - 新增 UI 方案文档
  - 替换导航按钮
  - 合并面板实现
  - 运行类型检查与构建验证
- 预计改动文件：
  - `src/game/scenes/Game.ts`
  - `.agents/docs/05-ui-ux/2026-04-17-character-panel-merge.md`
  - `.agents/docs/07-engine-tech/2026-04-15-features-tracking-migrated.md`
- 风险点与回滚方案：
  - 风险：面板内容过密导致重叠
  - 回滚：保留兼容方法，必要时可快速拆回双面板

## 6. 测试计划

- 手工测试步骤：
  - 点击“角色”按钮，确认打开统一面板
  - 检查左侧属性、右侧装备是否完整显示
  - 点击已装备物品，确认卸装功能正常
  - 悬浮装备确认 tooltip 正常
  - 分配属性点后确认面板刷新正常
- 边界情况：
  - 无装备时右侧槽位正常显示空状态
  - 无专精、无技能、无可分配点时面板排版正常
- 性能影响检查：
  - 面板打开与关闭不引入明显卡顿
