# 功能开发文档：狂战士主动技能特效接入

## 1. 基本信息

- 日期：2026-04-18
- 负责人：Codex
- 分类：05-ui-ux
- 关联功能：狂战士主动技能美术特效接入
- 状态：Testing

## 2. 目标与范围

- 目标：
  - 将 `public/assets/kuangzhanshi/skills` 下现有狂战士主动技能逐帧 PNG 接入游戏内播放。
  - 在技能释放时显示对应一次性特效，替代当前仅有伤害数字和 AOE 预警的表现。
- 本次范围（In Scope）：
  - 狂战士基础主动与转职后主动技能的素材配置、预加载、动画注册和运行时播放。
  - 根据技能类型区分近战斩击、扇形斩击、圆形爆发的落点、朝向和缩放。
  - 保持被动技能和触发技能现状不变。
- 非本次范围（Out of Scope）：
  - 被动技能与触发技能的新特效制作。
  - 游侠、法师技能特效接入。
  - 离线合成 spritesheet 资源。

## 3. 需求与验收标准

- 功能需求：
  1. `裂地斩`、`裂地震击`、`处刑突袭`、`血弧斩`、`战旗冲锋`、`战旗震荡`、`血怒反斩`、`猩红裂地` 可在释放时播放对应特效。
  2. 素材以逐帧 PNG 方式加载，不要求修改原始资源目录。
  3. 单体斩击类特效朝目标方向显示，AOE 类特效落在技能中心点。
  4. 特效播放一次后自动销毁，不残留对象。
- 验收标准（可检查）：
  - [ ] 狂战士 8 个主动技能均能看到对应美术特效。
  - [ ] 特效与伤害结算时机一致，不影响原有技能逻辑。
  - [ ] AOE 技能的特效位置与技能命中区域基本一致。
  - [ ] TypeScript 校验通过。

## 4. 方案设计

- 玩法/交互设计：
  - 单体斩击类技能在玩家与目标之间偏前方位置播放，并按攻击方向旋转。
  - 自身圆形 AOE 在玩家脚下播放，目标圆形 AOE 在目标点播放。
  - 扇形斩击沿玩家朝向播放，保留明显挥砍感。
- 技术设计：
  - 新增 `player-skill-visuals.ts` 管理逐帧 PNG 配置、加载 key、动画 key 和运行时表现参数。
  - `Preloader` 按配置逐帧加载狂战士技能图。
  - `Game` 中新增技能特效动画注册和播放方法，并在 `tryUseClassSkill` / `resolveAoeClassSkill` 成功释放时调用。
- 数据结构与配置：
  - 每个技能配置包含：`skillId`、素材目录、帧文件名模式、帧数、帧率、缩放、偏移和播放模式。
  - 支持数字序号帧和 `Explosion_n` 两种文件命名。
- 兼容性与迁移影响：
  - 仅新增美术表现层，不改动技能数值、存档或解锁逻辑。

## 5. 实现计划

- 任务拆分：
  1. 新增狂战士技能特效配置文件。
  2. 在预加载阶段逐帧加载素材。
  3. 在游戏场景中创建技能特效动画并补充播放逻辑。
  4. 为单体、扇形、圆形技能分别设置落点与朝向。
  5. 运行 TypeScript 校验。
- 预计改动文件：
  - 新增：`src/game/player-skill-visuals.ts`
  - 修改：`src/game/scenes/Preloader.ts`
  - 修改：`src/game/scenes/Game.ts`
  - 修改：`.agents/docs/07-engine-tech/2026-04-15-features-tracking-migrated.md`
- 风险点与回滚方案：
  - 风险：部分素材视觉中心与技能中心不一致，导致落点偏移。
  - 缓解：在配置中提供缩放和位移参数修正。
  - 回滚：移除技能特效播放调用，保留预警与伤害数字表现。

## 6. 测试计划

- 手工测试步骤：
  1. 使用狂战士进入战斗，验证基础主动 `裂地斩`。
  2. 切换各专精，验证单体主动和 AOE 主动各自特效。
  3. 验证 AOE 技能在多目标情况下仍只播放一次特效。
  4. 验证无目标、未命中或条件不满足时不会错误播放特效。
- 边界情况：
  - 技能击杀目标时特效仍正常播放并自动销毁。
  - 玩家朝向上下左右切换时，特效方向不反。
  - 场上多次连续施法时，不出现明显残影堆积。
- 性能影响检查：
  - 连续战斗下帧率无明显下降。
  - 播放后相关 sprite 能及时销毁。
