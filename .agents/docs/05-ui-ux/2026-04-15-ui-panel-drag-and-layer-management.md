# 功能开发文档：UI 面板拖拽与层级统一管理

- 日期：2026-04-15
- 负责人：Codex

## 2. 目标与范围

- 目标：实现 UI 面板可拖拽与边界限制，并建立统一层级管理，避免角色/怪物/战利品等渲染到面板之上。
- 本次范围（In Scope）：
  - 面板拖拽（基于标题栏拖拽手柄）
  - 面板边界限制（始终在可视区域内）
  - UI/Tooltip/GameObject 层级常量化管理
  - 面板显示逻辑统一封装（创建、动画、拖拽、边界）
- 非本次范围（Out of Scope）：
  - 多面板并行打开与层级栈管理
  - 面板布局重构与美术风格改版

## 3. 需求与验收标准

- 功能需求：
  - UI 面板可通过标题栏区域拖拽。
  - 拖拽过程中面板不会离开屏幕可视区域。
  - 打开背包等 UI 时，游戏内对象不得渲染到面板之上。
  - 新增面板时复用统一面板管理逻辑，减少重复与遗漏。
- 验收标准（可检查）：
  - [ ] 背包、装备、属性、消耗品、商店、确认弹窗、离线收益面板均可拖拽
  - [ ] 所有面板拖拽时保持在 `1024 x 768` 视口范围内
  - [ ] Tooltip 始终位于面板上层
  - [ ] 角色、怪物、掉落、浮字不会覆盖 UI 面板

## 4. 方案设计

- 玩法/交互设计：
  - 每个面板顶部增加可拖拽区域，拖拽时标题栏高亮，提供视觉反馈。
- 技术设计：
  - 在 `Game.ts` 中定义统一 `DEPTH` 常量。
  - 新增通用方法：
    - `createManagedPanel`
    - `enablePanelDragging`
    - `clampPanelPosition`
    - `createDragHandle`
  - 所有面板统一通过上述方法接入。
- 数据结构与配置：
  - `PanelRect`: `{ x, y, width, height }`
- 兼容性与迁移影响：
  - 不改动存档结构；仅影响 UI 行为与渲染层级。

## 5. 实现计划

- 任务拆分：
  - 增加层级常量与通用面板管理方法
  - 各面板接入统一创建/拖拽/边界逻辑
  - 调整 Tooltip 与世界对象层级
- 预计改动文件：
  - `src/game/scenes/Game.ts`
  - `.agents/docs/07-engine-tech/2026-04-15-features-tracking-migrated.md`
- 风险点与回滚方案：
  - 风险：拖拽事件与点击交互冲突。
  - 回滚：可临时禁用拖拽手柄，仅保留层级修复。

## 6. 测试计划

- 手工测试步骤：
  - 打开各面板，验证可拖拽、可关闭、边界限制生效。
  - 拖拽后尝试打开 Tooltip，验证覆盖层级正确。
  - 面板打开时观察角色/怪物/战利品/浮字是否覆盖 UI。
- 边界情况：
  - 快速连续开关面板；拖拽中点击关闭；窗口底部与右侧极限拖拽。
- 性能影响检查：
  - 拖拽过程无明显卡顿，Tween 与输入事件无泄漏。
