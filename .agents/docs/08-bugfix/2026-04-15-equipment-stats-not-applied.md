# 功能开发文档：装备主属性未正确应用到角色最终属性

## 1. 基本信息

- 日期：2026-04-15
- 负责人：Codex
- 分类：08-bugfix
- 关联功能：装备系统 / 角色最终属性计算 / 战斗攻速
- 状态：Testing

## 2. 目标与范围

- 目标：修复装备主属性只显示在 UI 中、未实际参与角色最终属性和战斗计算的问题。
- 本次范围（In Scope）：
  - 装备提供的 HP / ATK / DEF / 攻速 / 暴击 / 暴伤 / 移速 正确参与最终属性计算
  - 战斗攻击间隔由最终攻速驱动，而不是固定常量
  - 药水与自动药水基于最终最大生命值计算
  - 保持基础属性与装备加成解耦，避免换装污染 `baseStats`
- 非本次范围（Out of Scope）：
  - 新增装备锁定系统
  - 重做词条数值体系
  - 战斗平衡二次调整

## 3. 需求与验收标准

- 功能需求：
  - 装备穿上后，HUD、属性面板、战斗伤害、承伤、移动速度必须一致变化。
  - 攻速类装备和增益需影响实际攻击频率。
  - HP 类装备需影响回血、自动药水阈值和药水回复量上限判断。
- 验收标准（可检查）：
  - [x] 装备 ATK / DEF / HP 后，角色最终属性立即变化
  - [x] 装备攻速后，实际攻击频率提升
  - [x] 装备移速后，寻路/移动速度提升
  - [x] HP 装备会提升最终最大生命值，药水与自动药水判断同步正确
  - [x] 卸下装备后，角色最终属性恢复，不污染基础属性

## 4. 方案设计

- 玩法/交互设计：
  - 不改 UI 交互，仅修正最终属性口径。
- 技术设计：
  - 扩展 `getEffectiveStats`，支持注入装备/增益附加值。
  - 在 `Game.ts` 中统一通过当前装备与增益计算最终属性。
  - 战斗循环改为根据最终攻速动态计算攻击间隔。
  - 消耗品系统支持传入最终属性加成，保证 HP 相关判断一致。
- 数据结构与配置：
  - 新增 `ExternalStatBonuses` 作为运行时合成属性结构。
- 兼容性与迁移影响：
  - 不改存档结构；仅修正运行时计算逻辑。

## 5. 实现计划

- 任务拆分：
  - 扩展角色最终属性计算接口
  - 接入装备与增益加成
  - 修正战斗攻速计算
  - 修正药水与自动药水逻辑
- 预计改动文件：
  - `src/game/systems/character-system.ts`
  - `src/game/systems/combat-system.ts`
  - `src/game/systems/consumable-system.ts`
  - `src/game/scenes/Game.ts`
- 风险点与回滚方案：
  - 风险：修复后战斗节奏明显变化。
  - 回滚：保留最终属性合成逻辑，仅将攻速攻击间隔回退到固定值。

## 6. 测试计划

- 手工测试步骤：
  - 穿戴 ATK / DEF / HP / 攻速 / 移速 装备，观察 HUD 与实战表现。
  - 卸下装备后确认数值恢复。
  - 装备 HP 物品后使用药水，确认回复逻辑与最终最大生命值一致。
- 边界情况：
  - 高攻速叠加
  - 空装备状态
  - 切换装备时正处于战斗中
- 性能影响检查：
  - 最终属性每帧计算不应造成可见卡顿。
