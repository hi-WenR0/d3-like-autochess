# 功能开发文档：连击击杀未触发死亡清理

## 1. 基本信息

- 日期：2026-04-17
- 负责人：Codex
- 分类：08-bugfix
- 关联功能：战斗结算 / 怪物死亡清理
- 状态：Testing

## 2. 目标与范围

- 目标：修复怪物被打到 0 血后没有进入死亡流程，导致表现为“打怪不会死”的问题。
- 本次范围（In Scope）：
  - 修复普通攻击连击第二段击杀时的 `monsterKilled` 结果。
  - 目标选择跳过 0 血怪物，避免卡住战斗目标。
- 非本次范围（Out of Scope）：
  - 不调整怪物数值、掉落、经验或技能伤害。
  - 不重做死亡动画。

## 3. 需求与验收标准

- 功能需求：
  - 连击第二段击杀怪物时，应正常触发经验、金币、掉落、击杀计数和死亡动画。
  - 0 血怪物不应继续被选为战斗目标。
- 验收标准（可检查）：
  - [x] `playerAttackMonster()` 在连击第二段击杀时返回 `monsterKilled: true`。
  - [x] `findNearestMonster()` 过滤 `hp <= 0` 的怪物。
  - [x] `npx tsc --noEmit` 通过。
  - [x] `npm run build-nolog` 通过。

## 4. 方案设计

- 玩法/交互设计：
  - 玩家视角不改变攻击机制，只修正击杀结果。
- 技术设计：
  - 将普通攻击首次伤害的 `monsterKilled` 从 `const` 改为 `let`。
  - 连击第二段调用 `monsterTakeDamage()` 后，将返回值写回 `monsterKilled`。
  - `findNearestMonster()` 增加 `monster.stats.hp <= 0` 过滤。
- 数据结构与配置：
  - 不新增字段。
- 兼容性与迁移影响：
  - 不影响旧存档。

## 5. 实现计划

- 任务拆分：
  - 修复连击击杀结果。
  - 修复目标选择过滤。
  - 运行 TypeScript 与生产构建。
- 预计改动文件：
  - `src/game/systems/combat-system.ts`
  - `src/game/scenes/Game.ts`
  - `.agents/docs/08-bugfix/2026-04-17-combo-kill-not-cleaned-up.md`
- 风险点与回滚方案：
  - 风险：击杀流程被更早触发后暴露重复击杀路径。
  - 回滚：恢复连击第二段不写回击杀结果，并移除目标过滤。

## 6. 测试计划

- 手工测试步骤：
  - 使用兽王或带 `combo` 词条装备战斗。
  - 观察怪物被连击第二段打到 0 血时是否正常死亡、掉落并推进楼层击杀进度。
  - 用远程职业确认不会锁定 0 血怪。
- 边界情况：
  - 连击未触发。
  - 第一段攻击直接击杀。
  - 连击第二段击杀。
- 性能影响检查：
  - 仅新增一次简单 HP 判断，无明显性能影响。
