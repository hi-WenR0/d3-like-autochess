# 功能开发文档：消耗品面板滑动列表防溢出

## 1. 基本信息

- 日期：2026-04-17
- 负责人：Codex
- 分类：08-bugfix
- 关联功能：FEATURES / 消耗品系统与 UI 面板
- 状态：Testing

## 2. 目标与范围

- 目标：修复消耗品数量较多时，消耗品显示面板内容向下溢出的问题。
- 本次范围（In Scope）：
  - 消耗品列表改为可滚动列表
  - 列表只渲染当前可见行
  - 支持鼠标滚轮、上下按钮和滚动条跳转
  - 保持消耗品使用按钮与活跃增益显示可用
- 非本次范围（Out of Scope）：
  - 改动消耗品掉落、购买或堆叠规则
  - 新增消耗品筛选或排序功能
  - 重构通用 UI 列表组件

## 3. 需求与验收标准

- 功能需求：
  - 消耗品面板内列表不会超出面板边界
  - 消耗品多于可见行时可以滚动查看
  - 使用消耗品后刷新列表并保持滚动范围合法
- 验收标准（可检查）：
  - [ ] 多组消耗品存在时，列表行被限制在固定可视区域内
  - [ ] 鼠标滚轮可以上下滚动列表
  - [ ] 上下按钮可以逐行滚动
  - [ ] 滚动条可以跳转到列表位置
  - [ ] 使用消耗品后不会因滚动索引越界导致空白或报错

## 4. 方案设计

- 玩法/交互设计：
  - 面板保留现有宽高和拖拽行为。
  - 消耗品列表固定在面板中上部，活跃增益固定在底部区域。
  - 当内容超过可见行数时显示滚动条、上下按钮和当前位置文本。
- 技术设计：
  - 在 `Game` 场景内增加 `consumableScrollIndex` 记录列表起始行。
  - 打开面板时过滤 `count > 0` 的消耗品，按可见行数切片渲染。
  - 通过 `PhaserMath.Clamp` 保证滚动索引始终在合法范围内。
  - 交互触发后刷新面板，复用现有 `openConsumablePanel()` 重绘流程。
- 数据结构与配置：
  - 新增运行时 UI 状态字段，不写入存档。
- 兼容性与迁移影响：
  - 不改变旧存档结构。
  - 不影响消耗品系统和商店系统的数据接口。

## 5. 实现计划

- 任务拆分：
  - 新增消耗品列表滚动状态字段
  - 重写消耗品面板列表渲染为可见行切片
  - 接入滚轮、上下按钮和滚动条交互
  - 更新文档状态并运行 TypeScript 校验
- 预计改动文件：
  - `src/game/scenes/Game.ts`
  - `.agents/docs/08-bugfix/2026-04-17-consumable-panel-scroll-list-overflow.md`
  - `.agents/docs/07-engine-tech/2026-04-15-features-tracking-migrated.md`
- 风险点与回滚方案：
  - 风险：滚动交互与面板拖拽事件冲突。
  - 回滚：撤回滚动控件渲染，恢复原固定列表渲染。

## 6. 测试计划

- 手工测试步骤：
  - 准备超过可见行数的消耗品堆叠，打开消耗品面板。
  - 使用鼠标滚轮、上下按钮和滚动条切换列表位置。
  - 在不同滚动位置点击使用消耗品，确认刷新后列表仍可用。
- 边界情况：
  - 消耗品为空。
  - 消耗品数量少于或等于可见行数。
  - 最后一页不足一屏。
  - 使用最后一项后滚动索引需要回退。
- 性能影响检查：
  - 列表只渲染可见行，消耗品多时对象数量更少。
