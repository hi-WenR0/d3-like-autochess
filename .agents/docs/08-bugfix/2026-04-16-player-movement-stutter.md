# 功能开发文档：玩家移动卡顿修复

## 1. 基本信息

- 日期：2026-04-16
- 负责人：Codex
- 分类：08-bugfix
- 关联功能：FEATURES / 玩家方向键移动控制
- 状态：Testing

## 2. 目标与范围

- 目标：修复玩家方向键移动表现明显卡顿、而敌人追踪平滑的问题。
- 本次范围（In Scope）：
  - 将玩家移动从离散步进改为基于 `dt` 的逐帧平滑移动
  - 保留当前方向键与 45 度斜向移动输入逻辑
  - 保证斜向与直线移动速度一致
- 非本次范围（Out of Scope）：
  - 引入 Phaser 物理系统
  - 改动敌人移动逻辑
  - 增加角色朝向动画或冲刺系统

## 3. 需求与验收标准

- 功能需求：
  - 玩家按住方向键时应连续平滑移动，而不是定时跳格
  - 斜向移动应继续使用归一化，避免比直线更快
  - 切换方向时不应出现明显停顿或节奏断层
- 验收标准（可检查）：
  - [ ] 玩家单方向移动时无明显跳步卡顿
  - [ ] 玩家 45 度移动时保持平滑且速度与单方向一致
  - [ ] 快速切换方向时不会出现额外顿挫
  - [ ] 不破坏探索、战斗、地图边界与敌人追踪流程

## 4. 方案设计

- 玩法/交互设计：
  - 玩家移动观感与敌人持续追踪观感对齐
  - 不改动现有方向键交互方式
- 技术设计：
  - 移除固定步长 + 时间间隔驱动的玩家移动节流
  - 基于当前按键状态生成归一化方向向量
  - 每帧使用 `moveSpeed * dt` 计算位移距离
- 数据结构与配置：
  - 不新增存档字段
  - 保持角色 `moveSpeed` 语义为像素/秒
- 兼容性与迁移影响：
  - 仅修改运行时移动逻辑，对旧存档无影响

## 5. 实现计划

- 任务拆分：
  - 创建 bugfix 文档并登记到功能跟踪
  - 重构玩家移动更新逻辑为逐帧位移
  - 清理不再需要的步进常量与节流状态
  - 运行 TypeScript 校验
- 预计改动文件：
  - `src/game/scenes/Game.ts`
  - `.agents/docs/07-engine-tech/2026-04-15-features-tracking-migrated.md`
- 风险点与回滚方案：
  - 风险：逐帧模型接入后手感偏快或偏慢
  - 回滚：恢复到旧逻辑，或仅引入一个统一速度系数微调

## 6. 测试计划

- 手工测试步骤：
  - 按住单方向键，确认角色连续平滑移动
  - 按住两个正交方向键，确认 45 度移动平滑且速度一致
  - 在探索与战斗状态下都验证移动手感
- 边界情况：
  - 玩家贴近地图边缘时仍被边界限制
  - 同时按下相反方向键时输入抵消，不产生异常位移
- 性能影响检查：
  - 仅增加轻量逐帧向量计算，预期无明显性能问题
