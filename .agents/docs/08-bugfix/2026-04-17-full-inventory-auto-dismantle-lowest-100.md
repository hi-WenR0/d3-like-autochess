# 功能开发文档：背包满时自动分解最低等级 100 件

## 1. 基本信息

- 日期：2026-04-17
- 负责人：Codex
- 分类：08-bugfix
- 关联功能：FEATURES / 背包与拾取流程
- 状态：Testing

## 2. 目标与范围

- 目标：修复背包满时玩家因掉落拾取失败而卡在拾取循环、无法继续战斗的问题。
- 本次范围（In Scope）：
  - 背包满时自动分解等级最低的 100 件装备
  - 自动分解后继续当前拾取流程
  - 增加日志提示分解结果
- 非本次范围（Out of Scope）：
  - 改动手动分解 UI
  - 增加可配置阈值或设置项
  - 改动现有自动分解按稀有度规则

## 3. 需求与验收标准

- 功能需求：
  - 满包导致拾取失败时自动腾出空间
  - 分解目标按装备等级从低到高选 100 件
  - 自动分解后不再卡在拾取状态
- 验收标准（可检查）：
  - [ ] 背包满时会自动分解最低等级 100 件装备
  - [ ] 自动分解后当前掉落可继续尝试拾取
  - [ ] 玩家不会因满包掉落而无法继续战斗
  - [ ] 日志会显示自动分解数量与获得精华

## 4. 方案设计

- 玩法/交互设计：
  - 以自动兜底方式避免流程卡死，不增加玩家操作负担
  - 纯按等级最低优先，不区分稀有度
- 技术设计：
  - 在库存系统新增“分解最低等级前 N 件”接口
  - 在 `looting` 满包分支里调用该接口后重试当前拾取
- 数据结构与配置：
  - 不新增存档字段
  - 固定批量数为 100
- 兼容性与迁移影响：
  - 仅影响运行时拾取异常分支，对旧存档无影响

## 5. 实现计划

- 任务拆分：
  - 新增库存批量分解接口
  - 在拾取流程满包分支接入自动分解
  - 添加日志反馈
  - 运行 TypeScript 校验
- 预计改动文件：
  - `src/game/systems/inventory-system.ts`
  - `src/game/scenes/Game.ts`
- 风险点与回滚方案：
  - 风险：低等级但高稀有装备也会被自动分解
  - 回滚：撤回自动分解分支，恢复原有拾取失败逻辑

## 6. 测试计划

- 手工测试步骤：
  - 填满背包后击杀怪物掉装备，确认自动分解触发
  - 确认自动分解后当前掉落可继续进入背包
  - 确认战斗流程不再因满包卡死
- 边界情况：
  - 背包少于 100 件时，全部被分解
  - 多件同等级装备时分解顺序稳定
- 性能影响检查：
  - 仅在满包分支做一次排序与切片，预期无明显性能问题
