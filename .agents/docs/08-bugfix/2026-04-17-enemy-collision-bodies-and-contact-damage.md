# 功能开发文档：敌人碰撞体与玩家接触伤害

## 1. 基本信息

- 日期：2026-04-17
- 负责人：Codex
- 分类：08-bugfix
- 关联功能：敌人追击 / 玩家受击
- 状态：Done

## 2. 目标与范围

- 目标：避免敌人持续追击时最终全部重叠在玩家身上，同时允许玩家穿过敌人，并在接触期间每 0.2 秒受到 2 点伤害。
- 本次范围（In Scope）：
  - 接入 Phaser 4 Arcade Physics
  - 为所有敌人添加实体碰撞体，敌人之间互相分离
  - 为玩家添加 overlap 判定体，不阻挡穿怪
  - 增加玩家接触敌人的持续伤害
- 非本次范围（Out of Scope）：
  - 地图墙体碰撞
  - 击退、硬直、受击动画
  - 敌人 AI 路径规划重写

## 3. 需求与验收标准

- 功能需求：
  - 敌人之间不能再全部压到玩家同一点
  - 玩家仍可穿过敌人
  - 玩家与敌人接触时每 0.2 秒掉 2 点血
- 验收标准（可检查）：
  - [x] 多个敌人接近玩家时会围绕玩家分离，不再完全重叠
  - [x] 玩家按方向键可穿过敌人，不会被卡住
  - [x] 单个敌人持续接触时按 0.2 秒节奏对玩家造成 2 点伤害
  - [x] 敌人死亡后对应碰撞体会被清理

## 4. 方案设计

- 玩法/交互设计：
  - 敌人互相有实体碰撞，玩家与敌人只有重叠检测
- 技术设计：
  - 在 `GameConfig` 中启用 `Arcade Physics`
  - 玩家和敌人各自使用不可见 physics host 作为碰撞真值
  - 敌人使用速度推进，容器位置由 physics host 同步
  - 接触伤害使用每个敌人独立冷却表控制
- 数据结构与配置：
  - 不改存档
  - 接触伤害参数固定为 `2` 点 / `200ms`
- 兼容性与迁移影响：
  - 仅影响场景运行时移动与受击逻辑

## 5. 实现计划

- 任务拆分：
  - 开启 Arcade Physics
  - 为玩家和敌人创建 physics host
  - 将敌人追击改为速度驱动
  - 新增玩家接触伤害判定
  - 运行 TypeScript 与构建验证
- 预计改动文件：
  - `src/game/main.ts`
  - `src/game/scenes/Game.ts`
  - `.agents/docs/08-bugfix/2026-04-17-enemy-collision-bodies-and-contact-damage.md`
- 风险点与回滚方案：
  - 风险：接入物理系统后，敌人靠近玩家的堆叠形态会与旧逻辑不同
  - 回滚：移除 physics host，恢复手写坐标推进

## 6. 测试计划

- 手工测试步骤：
  - 玩家不动时观察多个敌人接近，确认不会完全重叠
  - 玩家穿过敌人时确认不会被阻挡
  - 保持与敌人接触，确认每 0.2 秒掉 2 点血
  - 击杀敌人后确认不会残留不可见碰撞体
- 边界情况：
  - 多个敌人同时接触玩家
  - 切层、死亡、回城后的碰撞体清理
- 性能影响检查：
  - 怪物数量有限，Arcade Physics 额外开销可接受
