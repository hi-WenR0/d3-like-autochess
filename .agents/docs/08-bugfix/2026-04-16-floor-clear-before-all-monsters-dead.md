# 功能开发文档：地牢未清空敌人时提前提示完成

## 1. 基本信息

- 日期：2026-04-16
- 负责人：Codex
- 分类：08-bugfix
- 关联功能：FEATURES / 地牢完成判定
- 状态：In Development

## 2. 目标与范围

- 目标：修复玩家尚未清空当前层敌人时，系统提前提示“已完成地牢”的问题。
- 本次范围（In Scope）：
  - 收紧楼层完成判定
  - 确保提示、自动存档和回城只在场上无存活怪物时触发
- 非本次范围（Out of Scope）：
  - 重构整套击杀计数系统
  - 改动怪物刷新规则

## 3. 需求与验收标准

- 功能需求：
  - 当前层仍有怪物存活时，不得提示完成。
  - 楼层完成提示必须同时满足计数达标和场上无怪物。
- 验收标准（可检查）：
  - [ ] 场上还有任意怪物时，不显示楼层完成面板
  - [ ] 自动存档不会在有残留怪物时触发
  - [ ] 清空全部怪物后，楼层完成流程正常触发

## 4. 方案设计

- 玩法/交互设计：
  - 玩家体验上只有在真正清图后才回城并看到完成提示。
- 技术设计：
  - 在场景层增加“当前层已真正清空”的辅助判定，要求 `monstersCleared >= monstersToClear` 且当前怪物容器为空。
- 数据结构与配置：
  - 不新增存档字段。
- 兼容性与迁移影响：
  - 仅收紧逻辑，不影响旧存档结构。

## 5. 实现计划

- 任务拆分：
  - 增加楼层清空辅助方法
  - 替换现有探索态与自动存档的完成判定
- 预计改动文件：
  - `src/game/scenes/Game.ts`
- 风险点与回滚方案：
  - 风险：若怪物容器未正确清理，可能导致无法结算。
  - 回滚：保留计数判定，同时针对异常怪物清理补日志。

## 6. 测试计划

- 手工测试步骤：
  - 留下一只怪物存活，确认不会提前提示完成。
  - 击杀最后一只怪物，确认正常进入完成流程。
- 边界情况：
  - 最后一只怪物通过技能击杀
  - 处于拾取阶段后立即完成本层
- 性能影响检查：
  - 仅增加常量级状态检查，不应有性能问题。
