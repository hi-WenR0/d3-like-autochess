# 功能开发文档：转职任务与条件解锁

## 1. 基本信息

- 日期：2026-04-16
- 负责人：Codex
- 分类：04-progression
- 关联功能：FEATURES / 转职任务与条件解锁
- 状态：Testing

## 2. 目标与范围

- 目标：为转职流程增加任务与条件限制，玩家需满足对应要求后才能完成转职，强化成长目标与职业身份感。
- 本次范围（In Scope）：
  - 转职前置条件定义
  - 转职任务目标与完成判定
  - UI 中展示当前转职条件进度
- 非本次范围（Out of Scope）：
  - 长链剧情任务系统
  - 复杂分支对话演出
  - 多职业多阶段重置转职

## 3. 需求与验收标准

- 功能需求：
  - 玩家需要满足指定条件后才能转职。
  - 转职条件可包括等级、击败怪物、完成地下层、拥有物品等。
  - 转职任务应与已有职业 / 专精系统衔接。
- 验收标准（可检查）：
  - [ ] 每个可转职方向都可配置前置条件
  - [ ] 条件未满足时不可转职，并明确提示缺失项
  - [ ] 条件满足后可进入转职确认或完成流程
  - [ ] 存档正确记录转职任务进度与完成状态
  - [ ] 与现有基础职业、专精选择流程兼容

## 4. 方案设计

- 玩法/交互设计：
  - 职业导师面板展示目标专精、前置条件和当前达成进度。
  - 条件满足后解锁“完成转职”操作。
- 技术设计：
  - 在职业配置中增加转职任务定义，支持多条件组合。
  - 将条件判断统一接入角色进度、图鉴、楼层、物品与击杀统计。
  - 与已有“职业系统扩展（二次转职与三专精）”文档协同，作为其解锁层补充。
- 数据结构与配置：
  - `AdvancementRequirement`
  - `AdvancementQuestState`
  - `requirements: level | floor | kill | item | unlock`
- 兼容性与迁移影响：
  - 旧存档已有转职状态时，任务进度可直接标记为已完成。

- 任务拆分：

- 风险点与回滚方案：
  - 风险：条件来源分散，导致状态同步复杂。
  - 回滚：先保留等级 + 楼层两类简单条件。

## 6. 测试计划

- 手工测试步骤：
  - 新角色查看转职条件，确认未满足时不可转职。
  - 达成指定条件后再次进入面板，确认状态更新。
  - 完成转职后存档并重新加载，验证状态持久化。
- 边界情况：
  - 条件刚满足与转职操作同帧发生
  - 已转职角色重复进入转职任务流程
  - 条件依赖已删除或已改名的怪物 / 物品配置
- 性能影响检查：
  - 条件刷新以事件驱动为主，避免每帧全量扫描。
