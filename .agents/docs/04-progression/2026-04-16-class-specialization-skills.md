# 功能开发文档：职业专精技能系统

## 1. 基本信息

- 日期：2026-04-16
- 负责人：Codex
- 分类：04-progression
- 关联功能：FEATURES / 职业专精技能系统
- 状态：Testing

## 2. 目标与范围

- 目标：围绕不同职业和专精设计独立技能体系，让职业成长不仅体现在属性上，也体现在战斗机制与技能选择上。
- 本次范围（In Scope）：
  - 基础职业与专精对应的技能池配置
  - 技能解锁条件与职业绑定
  - 技能对战斗系统的基础接入规划
- 非本次范围（Out of Scope）：
  - 全量技能特效与演出精修
  - 复杂技能树编辑器
  - PVP 或联机平衡方案

## 3. 需求与验收标准

- 功能需求：
  - 不同职业和专精应拥有不同技能。
  - 技能应与职业定位一致，如近战爆发、远程风筝、召唤、辅助等。
  - 转职与专精选择后，技能池应发生可见变化。
- 验收标准（可检查）：
  - [ ] 基础职业至少有一组专属技能定义
  - [ ] 每个专精都可配置额外技能或技能变体
  - [ ] 角色只可使用与当前职业 / 专精匹配的技能
  - [ ] 转职或切换专精后技能池正确刷新
  - [ ] 存档可正确保存已解锁技能状态

## 4. 方案设计

- 玩法/交互设计：
  - 技能围绕职业定位设计，例如狂战士偏近战爆发，游侠偏机动远程，法师偏元素和召唤。
  - 技能 UI 需要展示来源职业、解锁条件、冷却和核心效果。
- 技术设计：
  - 扩展技能配置，增加职业、专精和解锁条件字段。
  - 在战斗系统中按当前职业 / 专精动态装配技能池。
  - 与已有职业系统、转职条件和近战 / 远程属性共用统一配置层。
- 数据结构与配置：
  - `ClassSkillDefinition`
  - `requiredClass`
  - `requiredSpecialization`
  - `unlockCondition`
- 兼容性与迁移影响：
  - 旧存档没有技能池数据时，可按当前职业即时重建默认技能列表。

## 5. 实现计划

- 任务拆分：
  - 设计基础职业与专精技能配置结构
  - 接入技能池组装与解锁判断
  - 接入战斗系统和技能 UI
  - 增加读档兼容与回退逻辑
- 预计改动文件：
  - `src/game/models/character.ts`
  - `src/game/systems/character-system.ts`
  - `src/game/systems/combat-system.ts`
  - `src/game/scenes/Game.ts`
  - `src/game/systems/save-system.ts`
- 风险点与回滚方案：
  - 风险：技能系统范围过大，首版难以完整落地。
  - 回滚：先实现被动技能和少量主动技能，保留配置扩展位。

## 6. 测试计划

- 手工测试步骤：
  - 分别选择不同职业和专精，检查技能池变化。
  - 触发技能释放，验证效果、冷却和职业限制。
  - 存档后重新加载，确认技能解锁状态正确恢复。
- 边界情况：
  - 转职时当前技能栏中存在不再合法的技能
  - 未满足条件的专精技能提前显示
  - 技能配置缺失或引用错误
- 性能影响检查：
  - 技能池装配应在状态变更时完成，避免战斗中频繁重算。
