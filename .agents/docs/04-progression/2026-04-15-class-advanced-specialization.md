# 功能开发文档：职业系统扩展（二次转职与三专精）

## 1. 基本信息

- 日期：2026-04-15
- 负责人：Codex
- 分类：04-progression
- 关联功能：FEATURES.md / 职业系统扩展：二次转职与三专精
- 状态：Testing

## 2. 目标与范围

- 目标：在现有职业体系基础上，增加二次转职机制，并让每个基础职业拥有 3 个专精方向，形成明确的成长分支与玩法差异。
- 本次范围（In Scope）：
  - 基础职业：狂战士、游侠、法师
  - 二次转职入口与规则
  - 每个职业 3 个专精分支（共 9 个）
  - 专精带来的属性倾向、技能倾向、成长倾向定义
- 非本次范围（Out of Scope）：
  - 完整美术资源替换
  - 全量技能动画与音效精修
  - 联机/排行榜相关联动

## 3. 需求与验收标准

- 功能需求：
  - 玩家可在满足条件后进行二次转职。
  - 狂战士、游侠、法师各自拥有 3 个可选专精。
  - 专精选择后应改变角色构筑方向（技能/属性/战斗策略）。
  - UI 中可查看当前职业、专精、未解锁专精条件。
- 验收标准（可检查）：
  - [ ] 角色创建可选择基础职业（狂战士/游侠/法师）
  - [ ] 达成条件后可触发二次转职流程
  - [ ] 每个基础职业可见 3 个专精分支并可选择其一
  - [ ] 专精生效后，角色属性与技能池发生可见变化
  - [ ] 存档正确记录职业与专精状态

## 4. 方案设计

- 玩法/交互设计：
  - 在主城新增职业导师入口，满足等级/任务条件后开启二次转职。
  - 选择专精前展示分支定位、核心被动与代表技能。
- 技术设计：
  - 扩展角色模型：基础职业、专精、转职状态。
  - 扩展成长系统与技能系统：按专精注入差异化配置。
  - 在 UI 面板增加职业/专精展示与选择流程。
- 数据结构与配置：
  - 基础职业：Berserker / Ranger / Mage
  - 专精（暂定命名，可后续平衡调整）：
    - 狂战士：屠戮者、战吼统帅、血怒守卫
    - 游侠：神射手、陷阱大师、兽王猎手
    - 法师：元素术士、奥术学者、召唤先知
- 兼容性与迁移影响：
  - 老存档无专精数据时，默认标记为“未转职”并提供一次免费选择。

## 5. 实现计划

- 任务拆分：
  - 定义职业与专精枚举/配置
  - 增加二次转职条件判断与状态切换
  - 接入技能池与属性成长修正
  - 补齐 UI 与存档读写
- 预计改动文件：
  - `src/game/models/character.ts`
  - `src/game/systems/character-system.ts`
  - `src/game/systems/save-system.ts`
  - `src/game/ui/` 下职业相关面板
- 风险点与回滚方案：
  - 风险：专精平衡失衡、存档兼容问题。
  - 回滚：通过配置开关禁用专精加成并保留基础职业玩法。

## 6. 测试计划

- 手工测试步骤：
  - 新建角色选择职业 → 升级达成条件 → 执行转职 → 验证属性与技能变化 → 重启验证存档。
- 边界情况：
  - 未达条件强行点击转职、重复转职、切换场景中断转职流程。
- 性能影响检查：
  - 检查专精被动在战斗循环中是否引入明显帧耗时增长。
