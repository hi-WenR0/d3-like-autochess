# 功能开发文档：装备职业匹配与武器职业绑定

## 1. 基本信息

- 日期：2026-04-16
- 负责人：Codex
- 分类：04-progression
- 关联功能：FEATURES / 装备职业匹配与武器职业绑定
- 状态：Testing

## 2. 目标与范围

- 目标：为装备系统增加职业匹配规则，限制不同职业之间的武器和关键装备通用，强化职业识别度与构筑差异。
- 本次范围（In Scope）：
  - 武器按职业或职业组绑定
  - 装备栏与背包中显示职业适配信息
  - 不可用装备禁止穿戴或标记为不可激活
- 非本次范围（Out of Scope）：
  - 装备熟练度系统
  - 多职业共享武器精细白名单平衡
  - 装备洗练与职业转换服务

## 3. 需求与验收标准

- 功能需求：
  - 不同职业不能通用不匹配的武器。
  - 装备生成、展示、比较和穿戴流程都应识别职业限制。
  - 后续转职和专精系统可复用装备适配规则。
- 验收标准（可检查）：
  - [ ] 装备模型新增职业适配字段
  - [ ] 非匹配职业无法装备对应武器
  - [ ] 背包和 Tooltip 显示装备可用职业信息
  - [ ] 旧存档中的已装备非法武器可被正确纠正或降级处理
  - [ ] 不影响防具、饰品等非职业绑定装备的现有流程

## 4. 方案设计

- 玩法/交互设计：
  - 角色只能装备与职业匹配的武器；不匹配装备在背包中以灰态或红字提示。
  - 后续可扩展为“基础职业可用 / 专精限定 / 全职业通用”三级规则。
- 技术设计：
  - 为装备模板增加 `allowedClasses` 或 `allowedClassGroups` 字段。
  - 在装备、比较、自动拾取和存档加载时统一校验职业适配。
  - 为转职流程增加装备重校验，避免转职后装备非法保留。
- 数据结构与配置：
  - `allowedClasses: CharacterClass[]`
  - `isClassRestricted: boolean`
- 兼容性与迁移影响：
  - 旧装备默认视为全职业通用；后续逐步补齐配置。

## 5. 实现计划

- 任务拆分：
  - 扩展装备模型与生成配置
  - 增加穿戴校验与 UI 提示
  - 接入背包对比和拾取逻辑
  - 增加读档兼容与非法装备纠正
- 预计改动文件：
  - `src/game/models/inventory.ts`
  - `src/game/systems/inventory-system.ts`
  - `src/game/systems/character-system.ts`
  - `src/game/scenes/Game.ts`
  - `src/game/systems/save-system.ts`
- 风险点与回滚方案：
  - 风险：旧存档存在跨职业武器，读档后状态异常。
  - 回滚：先仅展示职业限制，不立即阻止装备。

## 6. 测试计划

- 手工测试步骤：
  - 使用不同职业拾取并尝试装备不同武器。
  - 观察背包、Tooltip、属性面板是否正确提示职业限制。
  - 转职后验证原武器是否仍合法。
- 边界情况：
  - 读档时已装备非法武器
  - 掉落生成未配置职业限制的武器
  - 多职业共享通用武器的显示与判定
- 性能影响检查：
  - 装备校验主要发生在状态变化时，不应造成持续性能负担。
