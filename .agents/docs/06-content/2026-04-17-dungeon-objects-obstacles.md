# 功能开发文档：Dungeon Objects 障碍物接入

## 1. 基本信息

- 日期：2026-04-17
- 负责人：Codex
- 分类：06-content
- 关联功能：战斗区静止障碍物
- 状态：In Development

## 2. 目标与范围

- 目标：在战斗区内随机生成 3 组来自 `Objects.png` 的静止障碍物，并为其添加碰撞体。
- 本次范围（In Scope）：
  - 使用 `Objects.png` 中指定 tile 组合生成 3 组障碍物
  - 每组在战斗区随机生成 1 个
  - 玩家和敌人都不能穿过这些障碍物
  - 碰撞体严格按用户指定的 tile 行放置
- 非本次范围（Out of Scope）：
  - 不接入更多 Objects 图块
  - 不做破坏、拾取、交互或动态障碍物
  - 不把障碍物写入存档

## 3. 需求与验收标准

- 功能需求：
  - 第一组：`127` 在上、`151` 在下，碰撞体在 `127`
  - 第二组：`128` 在上、`152` 在下，碰撞体在 `128`
  - 第三组：`42/43`、`66/67`、`90/91` 组成 `2x3` 结构，碰撞体在 `66/67`
  - 三组分别随机生成一个
- 验收标准：
  - 战斗区内可看到三组静止障碍物
  - 玩家移动时会被障碍物挡住
  - 敌人追击时不会穿过障碍物
  - 障碍物不会生成在战斗区边框之外

## 4. 方案设计

- 玩法/交互设计：
  - 障碍物只提供地形阻挡，不承担伤害或交互
- 技术设计：
  - 使用单独的 tilemap layer 渲染 `Objects.png` 图块
  - 使用不可见 Arcade physics host 作为碰撞体
  - 敌人与障碍物通过 collider 分离
  - 玩家移动前先做一次落点碰撞检测
- 数据结构与配置：
  - 障碍物布局运行时随机生成，不落盘
- 兼容性与迁移影响：
  - 不改现有地板和墙边铺法
  - 不改掉落、投射物和 HUD

## 5. 实现计划

- 任务拆分：
  - 新增开发文档
  - 添加障碍物渲染层与随机布局
  - 添加障碍物碰撞体与清理逻辑
  - 接入玩家/敌人阻挡
  - 运行类型检查与构建验证

## 6. 测试计划

- 手工测试步骤：
  - 进入战斗区确认三组障碍物可见
  - 玩家从四个方向接近障碍物，确认会被挡住
  - 敌人追击玩家时，确认会绕开或卡在障碍物外侧，不直接穿过
- 边界情况：
  - 障碍物不应和边框重叠
  - 障碍物不应全部生成在玩家初始点上
