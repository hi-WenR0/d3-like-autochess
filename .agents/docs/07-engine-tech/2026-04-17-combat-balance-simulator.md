# 功能开发文档：战斗数值模拟器与平衡调参

## 1. 基本信息

- 日期：2026-04-17
- 负责人：Codex
- 分类：07-engine-tech
- 关联功能：游戏平衡性调整、核心战斗系统优化、职业专精技能系统
- 状态：Implemented

## 2. 目标与范围

- 目标：解决当前游戏难度偏低、整体数值不平衡的问题，建立一个可重复运行的数值模拟器，用数据辅助调整怪物、角色、装备、技能和成长曲线。
- 本次范围（In Scope）：
  - 设计一个离线数值模拟器方案，首选 Python 脚本，也可后续导出 CSV 给 Excel / 表格工具分析。
  - 支持批量模拟不同楼层、职业、专精、装备强度、技能搭配下的战斗结果。
  - 输出关键平衡指标，用于定位“玩家过强”“怪物过弱”“成长曲线断层”等问题。
  - 形成可迭代调参流程：模拟、观察指标、调整配置、再次模拟。
- 非本次范围（Out of Scope）：
  - 不在首版直接引入复杂 AI、机器学习或自动寻优。
  - 不要求模拟器 100% 复刻 Phaser 场景表现，只对战斗和成长数值做近似验证。
  - 不直接改动现有战斗数值，调参应在模拟器跑通后分批进行。

## 3. 需求与验收标准

- 功能需求：
  - 可通过命令行运行模拟器，例如 `python scripts/balance_simulator.py`。
  - 可配置模拟参数：楼层范围、角色等级、职业 / 专精、装备稀有度、模拟次数、随机种子。
  - 可输出 CSV 或 Markdown 汇总，包含胜率、平均击杀时间、平均剩余血量、死亡楼层、每秒伤害、受到伤害、药水消耗等指标。
  - 可对比多组参数，观察调参前后的变化。
  - 模拟器逻辑应尽量复用或镜像现有 TypeScript 数据定义，减少与游戏真实逻辑偏离。
- 验收标准（可检查）：
  - [x] 首版脚本可在本地无额外服务依赖下运行 (`scripts/balance_simulator.py`)。
  - [x] 可批量模拟至少 1000 场战斗并输出稳定汇总。
  - [x] 可覆盖至少 3 个基础职业、9 个专精和多个楼层区间。
  - [x] 输出中能识别当前难度偏低的具体区间或来源。
  - [x] 调整一组怪物 / 成长参数后，模拟结果能展示前后差异（需手动运行两次比较）。

## 4. 方案设计

- 玩法/交互设计：
  - 模拟器不是玩家可见功能，而是开发调参工具。
  - 优先回答三个问题：
    - 玩家在每个楼层的预期胜率是多少？
    - 单场战斗时长是否过短，导致难度没有压迫感？
    - 装备、技能和职业成长是否让玩家过早碾压怪物？
- 技术设计：
  - 首版使用 Python 脚本实现，降低运行和数据分析成本。
  - 输入配置可先写在脚本内或 JSON / YAML 中，后续再从 TypeScript 配置导出。
  - 输出 CSV 便于 Excel、Numbers 或数据可视化工具读取。
  - 固定随机种子，保证同一配置下结果可复现。
- 数据结构与配置：
  - `SimulationScenario`：职业、专精、等级、楼层范围、装备档位、技能档位、模拟次数。
  - `CombatSnapshot`：角色有效属性、怪物属性、技能冷却、词条效果。
  - `SimulationResult`：胜率、平均 TTK、平均剩余 HP、死亡率、药水消耗、金币 / 经验收益。
- 兼容性与迁移影响：
  - 首版不影响游戏运行时代码。
  - 后续如需复用 TypeScript 配置，可增加导出脚本或共享 JSON 数据源。

## 5. 实现计划

- 阶段 1：定义指标和基线
  - 梳理当前角色、怪物、装备、技能、楼层成长公式。
  - 确定目标体验区间，例如普通层胜率、Boss 层胜率、平均战斗时长。
- 阶段 2：实现 Python 首版模拟器
  - 建立角色、怪物和战斗循环的轻量模型。
  - 支持楼层区间和多职业批量模拟。
  - 输出 CSV / Markdown 汇总。
- 阶段 3：跑当前版本基线
  - 记录当前数值下的难度曲线。
  - 标出明显过低难度区间和可能原因。
- 阶段 4：提出调参建议
  - 调整怪物生命、攻击、防御、成长倍率。
  - 调整装备词条、技能倍率和恢复来源。
  - 对比调参前后结果。
- 阶段 5：将确认后的参数回写游戏
  - 分批修改游戏配置。
  - 每批修改后重新运行模拟器和构建检查。

## 6. 测试计划

- 手工测试步骤：
  - 运行默认模拟场景，确认输出文件生成。
  - 修改随机种子，确认指标趋势稳定但单场结果可变化。
  - 调高怪物攻击或生命，确认胜率 / 战斗时长指标发生合理变化。
  - 使用 Excel 或表格工具打开 CSV，确认列名和数据格式可读。
- 边界情况：
  - 极低等级角色挑战高楼层。
  - 高装备档位角色挑战低楼层。
  - 高恢复 / 高吸血构筑导致战斗无限拖长。
  - 冷却缩减、复活、药水等保命效果叠加。
- 性能影响检查：
  - 模拟器离线运行，不影响游戏帧率。
  - 1000 场基础模拟应在可接受时间内完成。
