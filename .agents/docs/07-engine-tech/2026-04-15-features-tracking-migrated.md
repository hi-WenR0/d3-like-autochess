# FEATURES.md - Phaser 4 Demo 功能跟踪文档

本文档用于跟踪项目功能需求、开发进度和优先级。所有提出的功能都应在此记录，并在开发过程中更新状态。

## 项目概述

这是一个基于Phaser 4的地牢探索RPG游戏，包含以下核心系统：
- 角色系统和属性成长
- 装备系统和词条效果
- 战斗系统（攻击、暴击、连击等）
- 地牢探索和怪物生成
- 背包和物品管理系统
- 存档系统和离线收益
- UI面板系统（背包、装备、属性等）

## 功能状态定义

- **待定** (Pending)：已提出但尚未评估的功能
- **已规划** (Planned)：已评估并计划在后续版本中实现
- **开发中** (In Development)：正在开发中
- **测试中** (Testing)：开发完成，正在进行测试
- **已完成** (Completed)：已实现并测试通过
- **已废弃** (Deprecated)：不再计划实现的功能
- **阻塞中** (Blocked)：由于依赖或技术限制暂时无法开发

**状态**: 测试中
**优先级**: 高
**提出日期**: 2026-04-17
**目标版本**: v1.3.0
**依赖**: 核心战斗系统优化、职业专精技能系统、装备系统和词条效果

**描述**:
当前游戏整体难度偏低，数值曲线不平衡。需要建立一个离线数值模拟器，通过批量模拟不同楼层、职业、专精、装备强度和技能搭配下的战斗结果，辅助调节怪物成长、角色成长、装备词条、技能倍率和恢复来源。

**验收标准**:
- [x] 提供可本地运行的数值模拟器脚本 (`scripts/balance_simulator.py`)
- [x] 支持配置楼层范围、职业 / 专精、等级、装备档位、模拟次数和随机种子
- [x] 输出胜率、平均击杀时间、平均剩余血量、死亡率、药水消耗等指标（药水消耗暂未模拟）
- [x] 可导出 CSV 或 Markdown，便于在 Excel / 表格工具中分析
- [x] 能基于当前版本跑出基线结果，并指出难度偏低的主要区间或来源
- [x] 能对比调参前后的指标变化（需手动运行两次比较）

**技术实现说明**:
- 首版优先使用 Python 脚本实现，例如 `scripts/balance_simulator.py`
- 建立轻量战斗模型，近似复刻角色、怪物、装备词条、技能和恢复逻辑
- 使用固定随机种子保证结果可复现
- 输出 CSV / Markdown 汇总，后续可接入图表或表格模板

**进展记录**:
- 2026-04-17: 新功能需求登记
- 2026-04-17: 已创建开发文档 `.agents/docs/07-engine-tech/2026-04-17-combat-balance-simulator.md`
- 2026-04-17: 已实现 Python 模拟器脚本，支持批量模拟、CSV/Markdown 输出，并成功识别难度偏低区间

**备注**:
首版目标是建立可重复的调参依据，不直接追求自动寻优。确认模拟指标后，再分批回写游戏数值。

### 城镇商人商店与自动移动系统

**状态**: 已规划  
**优先级**: 中  
**提出日期**: 2026-04-17  
**目标版本**: v1.2.0  
**依赖**: 经济系统、移动控制系统  

**描述**:  
在主城添加商人NPC，提供能力购买服务。玩家可以购买“自动移动能力”和未来扩展的辅助功能，并可在游戏设置中切换自动/手动移动模式，增强游戏便利性和玩家自定义体验。

**验收标准**:  
- [ ] 主城场景中存在可见的商人NPC  
- [ ] 与商人交互可打开购买界面  
- [ ] 购买界面显示商品（自动移动能力）和价格  
- [ ] 玩家可用金币购买能力，购买后永久拥有  
- [ ] 游戏设置中提供自动/手动移动模式切换选项  
- [ ] 自动移动模式下角色能在地牢中自主移动探索  
- [ ] 手动移动模式下玩家使用方向键控制角色  
- [ ] 移动模式设置立即生效且可随时切换  
- [ ] 已购能力和移动模式设置可正确存档和加载  

**技术实现说明**:  
- 扩展角色数据模型：添加 `purchasedAbilities: string[]` 和 `movementMode: 'auto' | 'manual'`  
- 创建商人NPC和商店界面场景  
- 实现购买逻辑和金币扣除  
- 修改移动控制系统，根据模式选择控制源  
- 实现简单的自动移动算法（朝最近未探索区域或怪物移动）  
- 在设置面板中添加移动模式切换UI  

**进展记录**:  
- 2026-04-17: 新功能需求登记，已创建开发文档 `.agents/docs/03-economy/2026-04-17-town-merchant-auto-movement.md`  

**备注**:
这是商店系统的第一步，未来可扩展更多商品（如自动拾取、经验加成等）。自动移动算法首版采用简单探索逻辑，后续可根据反馈优化。

### 技能特效优化

**状态**: 已完成
**优先级**: 中
**提出日期**: 2026-04-17
**目标版本**: v1.2.0
**依赖**: 视觉效果系统、怪物技能系统

**描述**:
优化所有技能特效，调整粒子数量、大小、持续时间等参数。当前特效过于夸张，视觉冲击过强，需要在保持辨识度的前提下降低强度，提升整体视觉体验。

**验收标准**:
- [x] 粒子数量合理减少（普通怪12、精英18、稀有24、Boss35）
- [x] 粒子大小缩小约30%
- [x] 受击闪光持续时间缩短（100ms → 60ms）
- [x] 技能预警圈透明度降低
- [x] 特效不影响游戏性能（60fps稳定）
- [x] 视觉反馈仍然清晰可辨

**技术实现说明**:
- 修改 `PARTICLE_CONFIGS` 中所有粒子配置参数
- 调整 Game.ts 中受击闪光持续时间
- 调整技能预警圆环视觉效果
- 保持特效辨识度的同时降低视觉强度

**进展记录**:
- 2026-04-17: 新功能需求登记，已创建开发文档 `.agents/docs/05-ui-ux/2026-04-17-skill-visual-effects-optimization.md`
- 2026-04-17: 完成所有粒子参数调整和视觉效果优化

**备注**:
Boss特效可适当保留较高强度，普通怪特效大幅简化。后续可考虑添加"高/中/低"特效质量设置选项。

### 敌人死亡折叠动画

**状态**: 已完成
**优先级**: 中
**提出日期**: 2026-04-17
**目标版本**: v1.2.0
**依赖**: 视觉效果系统、怪物系统

**描述**:
改进敌人死亡视觉效果，让图像精灵在死亡时沿Y轴折叠消失。当前敌人死亡时直接销毁精灵，视觉上较为突兀，需要添加平滑的过渡动画。

**验收标准**:
- [x] 敌人死亡时身体精灵有 scaleY 折叠动画
- [x] 折叠持续时间约500ms
- [x] scaleY 从1.0缩小到0.1时销毁
- [x] 与粒子死亡效果配合正常
- [x] 各类型怪物（普通/精英/稀有/Boss）动画正常

**技术实现说明**:
- 在 `onMonsterKilled` 方法中添加折叠动画
- 使用 Phaser tween 实现 scaleY 动画
- 设置精灵原点到底部（origin.y = 1）
- 动画完成后销毁精灵和容器

**进展记录**:
- 2026-04-17: 新功能需求登记，已创建开发文档 `.agents/docs/05-ui-ux/2026-04-17-enemy-death-fold-animation.md`
- 2026-04-17: 完成实现，添加死亡折叠动画

**备注**:
折叠动画与粒子效果配合，产生"被击碎"的视觉感觉。动画时间较短（0.5秒），不影响游戏节奏。
